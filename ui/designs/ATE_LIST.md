## 화면 개요

LEFTOVERS 화면에서 [다먹음] 처리된 항목이 쌓이는 히스토리 화면.
사용자가 실수로 "다먹음" 처리한 경우 [덜먹음] 버튼으로 남은요리 상태로 복귀할 수 있다.
항목은 `eaten_at DESC` (최신순)으로 정렬되며, 30일 후 자동 숨김(`auto_hide_at`) 정책이 적용된다.

- **Screen ID**: `ATE_LIST`
- **권한**: 로그인 필수
- **API**: `GET /leftovers?status=eaten` (10-1) / `POST /leftovers/{id}/uneat` (10-3)
- **진입**: LEFTOVERS 화면에서 탭 전환 또는 네비게이션
- **이탈**: 뒤로가기 -> LEFTOVERS / 하단 탭 전환
- **DB 테이블**: `leftover_dishes` (status='eaten', eaten_at IS NOT NULL, auto_hide_at IS NOT NULL)

### Unauthorized 상태

비로그인 사용자가 직접 URL로 진입한 경우.
LEFTOVERS 자체가 로그인 필수이므로 일반적으로 도달하지 않지만,
세션 만료 등 예외 상황에서 보호한다.

```
+---------------------------------------+
|        로그인이 필요해요              |  <- 모달 오버레이
|                                       |     --panel 배경
|  다먹은 기록을 보려면                |     --radius-xl 상단
|  로그인이 필요해요.                  |
|                                       |
|    [로그인]        [취소]            |  <- [로그인]: --brand
|                                       |     [취소]: --text-2, ghost
+---------------------------------------+

로그인 성공 후: return-to-action -> ATE_LIST 자동 진입
```

## 320px 대응 (작은 모바일 sentinel)

- 행 내부 좌우 패딩: --space-4 (16px) -> --space-3 (12px) 축소
- 레시피명: text-base (16px) 유지, 1줄 말줄임 확보 (최대 폭 약 180px)
- 날짜: text-sm (13px) 유지
- [덜먹음] 버튼: 최소 44px 높이 유지, 폭은 auto (패딩 유지)
- 컨테이너 수평 여백: --space-4 (16px) -> --space-3 (12px)
- 행 간 간격: --space-2 (8px) 유지

320px에서 확인해야 할 blocker:
- [ ] 레시피명이 1줄 말줄임으로 표시되는지 (긴 이름 "소고기무국찌개" 등)
- [ ] [덜먹음] 버튼이 잘리지 않는지
- [ ] 날짜와 레시피명이 겹치지 않는지
- [ ] 터치 타겟 44px 유지 여부
- [ ] 행 높이가 과도하지 않은지 (컨텐츠에 비해)

## 접근성 노트

- **[덜먹음] 버튼**: `aria-label="레시피명 덜먹음 처리"` (예: "김치찌개 덜먹음 처리")
- **리스트 역할**: `role="list"` + 각 행 `role="listitem"`
- **날짜 표시**: 스크린 리더에서 "4월 25일에 다먹음 처리됨"으로 읽히도록 `aria-label` 보강
- **empty 상태**: 빈 리스트일 때 `aria-live="polite"`로 안내 문구 전달
- **행 제거 시**: 덜먹음 처리 후 "김치찌개가 남은요리로 돌아갔어요" 토스트를 `aria-live="assertive"`로 전달
- **색상 대비**: --foreground (#1a1a2e) on --surface (#ffffff) = 15.4:1 (AAA 통과), --brand (#ED7470) on white = 3.1:1 (버튼 텍스트로 사용 시 주의 -- outlined 버튼이므로 배경 대비 기준)
- **포커스 관리**: 행 제거 후 포커스를 다음 행의 [덜먹음] 버튼으로 이동

---

## 디자인 결정 사항

1. **행 레이아웃 -- 카드 vs 구분선 리스트**: 화면정의서 SS 16에는 구체적 레이아웃이 명시되어 있지 않다. LEFTOVERS(부모 화면)의 카드 패턴과 일관성을 맞추되, ATE_LIST는 정보 밀도가 낮으므로 (레시피명 + 날짜 + 버튼) compact한 카드 행으로 설계했다. 그림자를 --shadow-1(약한 그림자)로 두어 COOK_READY_LIST(--shadow-2)보다 시각적 무게를 낮춤. 이 화면은 히스토리성이므로 주요 행동 화면보다 조용해야 한다.

2. **[덜먹음] 버튼 스타일 -- outlined**: 다먹음을 되돌리는 "취소" 성격의 액션이므로 primary filled 대신 outlined 스타일을 사용한다. LEFTOVERS의 [다먹음] 버튼이 primary filled라면, 역방향 액션인 [덜먹음]은 시각적 무게를 낮추는 것이 자연스럽다.

3. **썸네일 미포함**: 화면정의서 SS 16에 "레시피명, 다먹은 날짜, [덜먹음]"만 명시되어 있다. 히스토리 성격의 간결한 리스트이므로 썸네일을 추가하지 않았다. LEFTOVERS(남은요리 목록)에서는 카드에 썸네일이 포함될 수 있지만, ATE_LIST는 "다먹은 기록" 조회용이므로 정보를 최소화한다.

4. **30일 자동 숨김 -- UI 미표시**: 정의서에 "30일 후 자동 숨김" 정책은 있지만, 사용자에게 남은 일수를 표시하라는 요구는 없다. 서버가 `auto_hide_at` 기준으로 필터링하며, UI에서는 단순히 보이지 않게 된다. 남은 일수 카운트다운이나 경고 표시는 임의 추가하지 않는다.

5. **LEFTOVERS와의 네비게이션 관계**: 유저플로우에서 `LEFTOVERS <-> ATE_LIST`는 양방향 관계다. ATE_LIST는 LEFTOVERS의 하위/동생 화면으로, 뒤로가기 시 LEFTOVERS로 복귀한다. 하단 탭은 "플래너" 활성 상태를 유지한다 (PLANNER_WEEK -> LEFTOVERS -> ATE_LIST 경로).

6. **Empty CTA 문구**: 정의서에 empty 문구는 "다먹은 기록이 없어요"로 명시되어 있다. 보조 설명과 CTA는 사용자가 다음 행동을 알 수 있도록 추가했다. [남은요리 보기] CTA로 LEFTOVERS 화면 이동을 유도한다.

---

## Anchor Screen 영향 분석

- **PLANNER_WEEK 의존**: 이 화면은 PLANNER_WEEK -> [남은요리] -> LEFTOVERS -> ATE_LIST 경로로 진입한다.
- **Anchor extension 해당 여부**: 해당하지 않음. ATE_LIST는 PLANNER_WEEK의 핵심 CTA를 변경하지 않으며, 남은요리 관리 여정의 말단 화면이다.
- **모바일 UX 리스크**: 낮음. 단순 리스트 화면으로, 가로 스크롤 없음, 모달 오버레이 없음, 단일 방향 액션([덜먹음])만 존재.
