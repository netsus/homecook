## 컴포넌트 상세

### 레시피 카드

**기본 상태 (체크 ON)**

- 왼쪽 체크박스: `--olive` 활성, 터치 타겟 `44×44px`
- 레시피명: `text-base` (16px), `--foreground`, 1행 wrap 허용
- 기본 인분 표시: 우측 정렬, `text-sm` (13px), `--muted`
- 인분 조정 스테퍼:
  - `-` 버튼: `44×44px`, `--olive` 테두리, 최소값 1
  - 숫자 표시: `text-lg` (18px), `--foreground`, 중앙 정렬
  - `+` 버튼: `44×44px`, `--olive` 테두리, 최대값 제한 없음
- 보조 정보 (합산 계획 인분): `text-sm` (13px), `--muted`, 왼쪽 정렬

**체크 OFF 상태**

- 카드 전체 opacity 0.6
- 인분 조정 스테퍼 disabled (interaction 불가)

**Loading 상태**

- 스켈레톤 높이: `80px`
- 배경: `--line` + pulse animation

**Empty**

- 카드 목록 전체가 비는 상태 → Empty 화면 노출

**Error**

- 카드 목록 fetch 실패 → Error 화면 노출

## 디자인 결정 사항

### 1. 레시피 카드 레이아웃

**결정**: 체크박스 + 레시피명 + 인분 조정 스테퍼를 한 카드 안에 세로로 배치

**이유**:
- 가로 공간이 제한된 모바일에서 체크박스와 스테퍼를 같은 행에 두면 레시피명 truncate 발생
- 세로 배치로 레시피명 1행 wrap 허용, 가독성 유지
- 카드당 높이 증가 trade-off는 허용 가능 (보통 5~10개 내외 레시피)

### 2. 인분 조정 스테퍼 위치

**결정**: 레시피명 아래 별도 행에 배치, 왼쪽 정렬

**이유**:
- 터치 타겟 `44×44px` 확보
- 스테퍼가 레시피명 우측에 붙으면 좁은 폭에서 레시피명 축소 위험
- 보조 정보(합산 계획 인분)와 시각적으로 구분 명확

### 3. Empty 상태 CTA

**결정**: [플래너로 돌아가기] → `PLANNER_WEEK` 복귀

**이유**:
- 장보기 대상이 없는 경우 사용자가 할 수 있는 유일한 행동은 플래너로 돌아가 식사 등록
- `MENU_ADD` 직접 진입보다 `PLANNER_WEEK` 복귀가 맥락상 자연스러움

### 4. 토큰 적용: `olive base + thin orange highlight`

**결정**: 체크박스, 스테퍼 테두리에 `--olive` 사용, `--brand`는 하단 CTA만 사용

**이유**:
- design-tokens.md § v1.5.0 방향: `olive base + thin orange highlight`
- `--olive`는 태그, 재료 필터, 서브 액션에 사용
- `--brand`는 주요 CTA에만 집중 (여기서는 [장보기 목록 만들기])

### 5. 화면 제목: "장보기 준비"

**결정**: 앱바 제목을 "장보기 준비"로 표기

**이유**:
- 아직 리스트가 생성되지 않은 준비 단계임을 명확히 전달
- `SHOPPING_DETAIL`의 "장보기 목록"과 구분

## design-critic 검토 필요 항목

- [ ] 레시피 카드 세로 배치가 5개 이상 리스트에서 스크롤 피로도 발생하는지 확인
- [ ] 인분 조정 스테퍼 터치 영역이 체크박스와 충분히 분리되는지 확인
- [ ] Empty 상태 "장보기 대상이 없어요" 문구가 충분히 명확한지 검토
- [ ] 좁은 폭(320px)에서 레시피명 1행 wrap이 카드 높이를 과도하게 늘리는지 확인
- [ ] [장보기 목록 만들기] 버튼이 하단 고정 시 safe-area-inset-bottom 반영 필요 여부 확인
- [ ] Loading 상태에서 "팬트리 재료를 확인 중이에요" 같은 세부 진행 상태 표시 필요 여부 검토
- [ ] 체크 OFF 상태 카드 opacity 0.6이 충분히 구분되는지 확인

## Scroll Containment 명시

- **세로 스크롤**: 레시피 카드 리스트 영역만 세로 스크롤 (앱바, helper copy, 하단 CTA는 고정)
- **가로 스크롤**: 없음
- **하단 CTA**: 화면 하단 고정 (스크롤 무관)

## Mobile UX Blocker 체크

| 규칙 | 준수 여부 | 비고 |
|------|----------|------|
