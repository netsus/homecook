## 인터랙션 플로우

### 체크 토글 (구매 완료 표시)

1. 사용자가 항목 체크박스 클릭
2. `PATCH /shopping/lists/{list_id}/items/{item_id}` 호출 (`is_checked: true/false`)
3. 성공 시 UI 즉시 반영 (낙관적 업데이트)
4. 실패 시 원래 상태로 롤백 + 에러 토스트

### 팬트리 제외 섹션으로 이동 (exclude→uncheck 규칙)

1. 사용자가 [팬트리 제외] 버튼 클릭
2. `PATCH /shopping/lists/{list_id}/items/{item_id}` 호출 (`is_pantry_excluded: true`)
3. **서버가 `is_checked=false`로 자동 정리** (exclude→uncheck 규칙)
4. 성공 시 해당 항목이 제외 섹션으로 이동 (애니메이션 선택)
5. 실패 시 에러 토스트

### 되살리기 (제외 섹션 → 구매 섹션)

1. 사용자가 [되살리기] 버튼 클릭
2. `PATCH /shopping/lists/{list_id}/items/{item_id}` 호출 (`is_pantry_excluded: false`)
3. 성공 시 해당 항목이 구매 섹션으로 이동
4. 실패 시 에러 토스트

### Read-only 모드에서 수정 시도 시

1. 사용자가 체크박스 또는 버튼 클릭 시도
2. UI에서 interaction 자체를 차단 (비활성 상태)
3. 만약 API가 호출되면 서버가 409 CONFLICT 반환
4. 에러 토스트: "완료된 장보기 기록은 수정할 수 없어요"

## 상태 관리

### 로컬 상태

- `isLoading`: 리스트 조회 중
- `items`: 전체 항목 배열 (`is_pantry_excluded` 기준으로 섹션 분리)
- `isReadOnly`: `is_completed` 기준으로 read-only 모드 판정

### 파생 상태

- `purchaseItems`: `items.filter(item => !item.is_pantry_excluded)`
- `excludedItems`: `items.filter(item => item.is_pantry_excluded)`
- `isEmpty`: `purchaseItems.length === 0`

## 접근성 (a11y)

- 체크박스: `role="checkbox"`, `aria-checked`, `aria-label="재료명 구매 완료 표시"`
- [팬트리 제외] / [되살리기] 버튼: `aria-label="재료명 팬트리 제외"`
- Read-only 모드: `aria-disabled="true"`, 포커스 차단
- 섹션 헤더: `role="heading"`, `aria-level="2"`

## 성능 고려사항

- 항목 목록: 가상 스크롤 불필요 (예상 항목 수 < 50개)
- 낙관적 업데이트: 체크 토글 시 즉시 UI 반영, 실패 시 롤백
- 정렬 인덱스: `(shopping_list_id, sort_order, id)` (DB 설계 v1.3.1 반영)
