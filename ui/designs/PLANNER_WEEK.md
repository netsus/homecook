## 핵심 컴포넌트

### 1. 상단 CTA 그룹

- `[장보기] [요리하기] [남은요리]` 3개 버튼을 항상 노출한다.
- 이번 슬라이스에서는 모두 `disabled` 상태다.
- 배경 `--surface`, 텍스트 `--muted`, border `--line`을 사용한다.
- 클릭/탭/키보드 활성화 시 아무 라우팅도 일어나지 않는다.

### 2. 주간 그리드

- 세로축은 날짜, 가로축은 끼니 컬럼이다.
- 모바일 기준 가로 스크롤 가능 구조를 허용한다.
- 컬럼 수는 기본 3개(아침/점심/저녁)에서 시작하며 최대 5개까지 확장한다.

### 3. 컬럼 관리

- `+` 컬럼은 새 컬럼 추가 트리거다.
- 각 컬럼 헤더에는 이름, 편집 진입 affordance, 삭제 affordance를 둔다.
- 삭제 가능 여부는 서버 응답으로 최종 확정한다.

### 4. Meal 셀

- 셀에는 레시피 제목, 계획 인분, 상태 뱃지만 노출한다.
- `is_leftover=true`인 meal은 배경 또는 accent로 별도 시각 강조를 준다.
- 이번 슬라이스에서는 셀 탭 이동을 연결하지 않는다.

### 5. 상태 뱃지

| 상태 | 라벨 | 색상 방향 |
| --- | --- | --- |
| `registered` | 식사 등록 완료 | `--brand` 계열 |
| `shopping_done` | 장보기 완료 | `--olive` 계열 |
| `cook_done` | 요리 완료 | 완료 강조색 |

- 색상만으로 구분하지 않고 텍스트 라벨을 함께 유지한다.

## 상호작용 규칙

- 컬럼 추가: 최대 5개 도달 시 `409` 에러 메시지를 안내한다.
- 컬럼 수정: 저장 즉시 헤더에 반영한다.
- 컬럼 삭제: 소속 meal 존재 시 삭제 불가 메시지를 표시한다.
- 날짜 이동: 위/아래 무한 스크롤로 범위를 갱신하고, 범위 변경 중에는 loading UI를 보여준다.

---

## 접근성 / 토큰 메모

- 페이지 배경은 `--background`, 카드/그리드 셀은 `--surface`를 사용한다.
- 기본 수평 여백은 모바일 `--space-4`.
- 터치 타겟 최소 크기 `44x44`.
- disabled CTA는 활성 버튼처럼 오인되지 않게 pointer affordance를 제거한다.
