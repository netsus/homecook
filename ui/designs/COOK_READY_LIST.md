## 화면 개요

PLANNER_WEEK 상단 [요리하기] 버튼에서 진입하는 화면.
오늘~마지막 등록일 범위의 `shopping_done` 상태 식사를 레시피별로 합산하여 리스트로 보여주고,
사용자가 레시피를 골라 요리 세션을 시작(COOK_MODE 이동)하거나 뒤로 돌아갈 수 있다.

- **권한**: 로그인 필수
- **API**: `GET /cooking/ready`
- **진입**: PLANNER_WEEK 상단 [요리하기]
- **이탈**: 뒤로가기 -> PLANNER_WEEK / [요리하기] -> COOK_MODE

## 320px 대응 (작은 모바일 sentinel)

- 카드 내부 패딩을 --space-3 (12px)에서 10px로 축소 가능
- 썸네일 크기: 64x64 -> 52x52 축소
- 레시피명: text-lg (18px) 유지, 1줄 말줄임 확보
- [요리하기] 버튼: 최소 44px 높이 유지, 폭은 auto
- helper section: 좌우 여백 --space-3 (12px)으로 축소
- 수평 여백: --space-4 (16px) -> --space-3 (12px)

320px에서 확인해야 할 blocker:
- [ ] CTA 버튼이 잘리지 않는지
- [ ] 레시피명이 1줄 말줄임으로 표시되는지
- [ ] 썸네일 + 텍스트 + 버튼이 한 카드에 수용되는지
- [ ] 터치 타겟 44px 유지 여부

## 디자인 결정 사항

1. **썸네일 포함 여부**: 화면정의서 SS 13에는 "레시피명, 합산 인분, [요리하기]"만 명시되어 있으나, API 응답(`recipe_thumbnail_url`)에 썸네일이 있고, SHOPPING_FLOW 등 유사 리스트 화면에서 카드에 시각적 앵커를 제공하기 위해 썸네일을 포함했다. 정의서에 "썸네일 불포함"이라는 명시적 제한은 없다.

2. **카드 레이아웃 방향**: 정의서에 레이아웃 구체 규격은 없다. SHOPPING_FLOW(유사 리스트 화면)와 PANTRY 화면의 카드 패턴을 참조하여, 썸네일 좌측 + 정보 우측 + CTA 우측 하단 배치로 결정했다. prototype-derived design 분류에 따라 Baemin vocabulary를 적용한다.

3. **date_range 표시**: API 응답에 `date_range`가 포함되므로, 사용자에게 대상 기간을 안내하는 helper copy로 활용한다. 화면정의서에 명시적 언급은 없지만, SHOPPING_FLOW의 범위 표시 패턴과 일관성을 맞춘다.

4. **하단 탭바 노출**: COOK_READY_LIST는 PLANNER_WEEK의 하위 화면이지만, full-page route이므로 하단 탭바를 유지한다. COOK_MODE(전체화면 몰입형)와 달리 일반 navigation 흐름이다.

5. **Empty CTA 문구**: 정의서에 empty 문구 규격이 없으므로, "장보기 완료된 레시피가 없어요" + "플래너에서 장보기를 먼저 완료해 주세요"로 설정했다. PLANNER_WEEK 복귀 유도는 workpack README에 명시된 요구사항이다.

6. **인분 표시 -- 읽기 전용 명확화**: 화면정의서 SS 13에 "합산 인분(읽기)"로 명시되어 있다. 스테퍼나 수정 UI는 의도적으로 배제한다. 요리모드 인분 조절 불가 정책(요구사항 2-5)과 일관된다.

## Anchor Screen 영향 분석

- **PLANNER_WEEK 의존**: 이 화면은 PLANNER_WEEK 상단 [요리하기] CTA에서 진입한다.
- **Anchor extension 해당 여부**: 해당하지 않음. PLANNER_WEEK에 이미 [요리하기] 버튼이 존재하고, 이 슬라이스는 그 버튼의 목적지 화면을 구현하는 것이다 (workpack README 참조). 핵심 CTA 추가/변경이 아니다.
- **모바일 UX 리스크**: 낮음. 단순 리스트 화면으로, 가로 스크롤 없음, 모달 오버레이 없음, 단방향 CTA.

## design-critic 검토 필요 항목

- [ ] 카드 내 썸네일 크기 (64px)가 320px에서도 레시피명/CTA와 균형을 유지하는지
- [ ] [요리하기] 버튼 위치가 thumb-friendly zone (우측 하단)에 적합한지
- [ ] 세션 생성 중 loading 인터랙션 (버튼 disabled + spinner)이 이중 탭을 충분히 방지하는지
- [ ] empty 상태에서 "플래너로 돌아가기" CTA가 충분히 발견 가능한지 (중앙 배치 확인)
- [ ] helper copy의 날짜 범위 표시가 긴 범위에서 줄바꿈 없이 표시되는지
- [ ] COOK_MODE에서 복귀 시 리스트 갱신 타이밍 (stale data 방지)
- [ ] 카드가 1개만 있을 때의 화면 밀도 -- 여백이 과하지 않은지
