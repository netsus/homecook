## 체크리스트 결과

### A. 요구사항 정합성

- [x] 화면정의서 §3의 모든 컴포넌트(미디어, 제목+태그, 지표, 인분조절, 버튼5개, 재료리스트, 스텝리스트)가 와이어프레임에 포함됐는가
- [x] 문서에 없는 컴포넌트/필드/기능이 추가됐는가 — 해당 없음. 플로팅 헤더 전환(Intersection Observer) 패턴은 와이어프레임 구현 방식이며 문서 위반 아님
- [x] 로그인 게이트 대상 액션(좋아요/저장/플래너 추가)이 모두 처리됐는가 — 3개 모두 "비로그인 → 로그인 게이트 모달" 처리
- [x] read-only 상태가 필요한 화면에서 올바르게 반영됐는가 — RECIPE_DETAIL은 read-only 대상 화면이 아님
- [x] 삭제된 엔드포인트 `DELETE /recipes/{id}/save`가 UI에 등장하지 않는가 — 등장하지 않음. 저장 액션은 `POST /recipes/{id}/save`(2-3), 저장 해제는 설계 범위 밖

### B. 공통 상태 커버리지

- [x] Loading 상태 (스켈레톤/인디케이터) 포함 — 전체 화면 스켈레톤 상세히 명시됨
- [x] Empty 상태 (안내 + CTA) 포함 — 재료/스텝 Empty 포함. 화면 자체 Empty는 해당 없음(레시피 ID가 있어야 진입 가능)
- [x] Error 상태 (안내 + [다시 시도]) 포함 — "레시피를 불러오지 못했어요" + [다시 시도] 명시
- [x] read-only 상태 — 이 화면에서는 해당 없음 (RECIPE_DETAIL은 read-only 대상 화면 아님)
- [◻] unauthorized 상태 — 비로그인 시 버튼 표시(시각적 동일) + 탭 시 게이트 처리로 설계되어 있으나, 존재하지 않는 레시피 ID 접근 시의 404 상태가 명시되어 있지 않다. Error 상태와 통합 처리로 볼 수 있으나 별도 명시가 없음

## 상세 분석 노트

### 잘 처리된 부분

1. **상태 커버리지 충실**: Loading(스켈레톤), Error(다시시도), Empty(재료/스텝 개별), 비로그인 시각 처리가 모두 명시되어 있고 각 컴포넌트 단위로 상태가 기술되어 있다.

2. **로그인 게이트 return-to-action**: `planner_add`, `like`, `save` 세 가지 액션 모두 returnToAction 파라미터가 명시되고 로그인 성공 후 자동 실행까지 연결되어 있다. 유저 Flow맵 ② return-to-action 대상 액션 테이블과 일치한다.

3. **인분 조절 정책 정확 반영**: ServingStepper가 레시피 상세에서만 활성화되고 COOK_MODE 진입 시 확정값을 전달하는 구조가 명확하다. 요구사항 2-5 인분 정책과 일치한다.

4. **레시피북 저장 타입 필터**: SavePopup에서 `book_type === 'saved' || 'custom'`만 노출하고 `my_added`, `liked` 제외를 명시했다. 요구사항 2-13 레시피북 정책과 일치한다.

5. **삭제 엔드포인트 미사용**: `DELETE /recipes/{id}/save`(2-4 삭제됨)가 설계 어디에도 등장하지 않는다. 저장 액션은 올바르게 `POST /recipes/{id}/save`(2-3)를 사용한다.

6. **독립 요리 상태 분리**: `mode='standalone'` 파라미터 전달로 독립 요리 경로를 명시하고, 화면정의서 매핑표에 "플래너 식사 상태 변경은 수행하지 않음"을 기록했다. AGENTS.md Domain Rules "독립 요리는 meals 상태를 바꾸지 않는다"와 일치한다.

7. **디자인 토큰 전반 준수**: C2 명랑한 주방 토큰(`--brand`, `--olive`, `--surface`, `--panel`, `--muted`, `--line`, `--foreground`)이 설계 전체에 일관되게 사용되었다. 조리방법 배지 색상도 design-tokens.md 테이블과 정확히 일치한다.

### 미디어 Empty 배경색 (`#e5e5e5`) 토큰 위반 경미한 확인

미디어 영역 Empty 상태 배경으로 `#e5e5e5`를 사용했는데, 이는 design-tokens.md에 정의되지 않은 임의 색상이다. `--line` 토큰이 `rgba(0,0,0,0.07)`로 매우 연하여 빈 미디어 영역 표현이 어렵다면, `--muted (#999999)` opacity-20 정도로 표현하거나 별도 토큰 추가를 요청하는 것이 바람직하다.
