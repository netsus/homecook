## 상세 검토 의견

### 강점

1. **요구사항 정합성 우수**
   - 화면정의서 §14의 모든 컴포넌트가 빠짐없이 포함됨
   - 조리 인분 읽기 전용, 좌우 스와이프, 소진 재료 팝업 등 핵심 정책 정확히 반영
   - 플래너 경유 / 독립 요리 시나리오 구분 명확

2. **상태 커버리지 완벽**
   - Loading(skeleton/스피너), Error(재시도/복귀), read-only(인분 고정) 모두 명시
   - Empty 상태 N/A 판단 적절 (세션 기반 진입)

3. **UX 품질 높음**
   - 전체화면 몰입형 + 최소 UI 원칙 명확
   - scroll containment 분명 (재료/스텝 각각 세로 스크롤, 좌우 스와이프는 화면 전환)
   - 터치 타겟 44px 보장, safe-area 대응 명시
   - 작은 모바일 sentinel(320px) 대응 전략 제시

4. **디자인 토큰 준수**
   - 최신 `--brand (#ED7470)` 값 사용 (2026-04-27 사용자 승인)
   - `--surface`, `--muted`, `--radius-md`, `--space-4` 등 토큰 일관성 유지
   - 조리방법 색상은 design-tokens.md 공식 토큰 사용

5. **도메인 규칙 정확**
   - 독립 요리와 플래너 요리의 상태 전이 차이 명확히 구분
   - 요리모드에서 인분 조절 불가 원칙 준수
   - 팬트리 소진 처리 (체크박스 기반, 수량 입력 없음) 정확

### 개선 제안 (마이너)

1. **소진 재료 팝업 스크롤 처리**
   - 현재: 재료 목록 표시만 명시
   - 제안: 재료가 많을 경우 팝업 내부 스크롤 명시 (max-height 60vh, overflow-y: auto)
   - 이유: 재료가 10개 이상일 경우 팝업이 화면을 넘어설 수 있음

2. **320px 좁은 폭 CTA 배치 확정**
   - 현재: "CTA 버튼 세로 배치 검토 (공간 부족 시)" (L255)
   - 제안: 320px 기준으로 세로 배치를 기본값으로 확정하거나, 가로 배치 유지 시 최소 폭 명시
   - 이유: 구현 단계에서 판단 부담 줄이기

3. **스와이프 affordance 우선순위**
   - 현재: 화살표/dot/텍스트 3가지 선택지 나열 (L229~231, L259~263)
   - 제안: "dot indicator + 작은 좌우 화살표 힌트" 조합을 기본안으로 추천
   - 이유: 첫 사용자 대상 affordance는 구현 단계 A/B 테스트 또는 사용자 피드백 기반 결정 권장

4. **조리방법 색상 표현 방식 확정**
   - 현재: "좌측 4px 세로 테두리 또는 상단 1줄 배경 tint" 선택지 (L225~227, L259)
   - 제안: 좌측 4px 세로 테두리를 기본안으로 확정
   - 이유: 모바일에서 좌측 테두리가 시각적 구분과 정보 밀도 유지에 더 유리

1. **우선순위 1 (필수)**

   - 스와이프 affordance (dot indicator + 화살표 힌트)
   - 조리방법 색상 좌측 4px 세로 테두리로 확정

3. **우선순위 3 (선택)**
