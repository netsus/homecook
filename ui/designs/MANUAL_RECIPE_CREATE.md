## 컴포넌트 상세

### AppBar (상단 앱바)
- **기본 상태**:
  - 왼쪽: 뒤로 버튼 (MENU_ADD 복귀)
  - 중앙: "직접 레시피 등록" (text-xl, weight-800, --foreground)
  - 오른쪽: [저장] 버튼 (--brand 색상, text-base, weight-600)
- **비활성**: 필수 필드 미입력 시 [저장] 버튼 disabled (--text-4 색상)
- **토큰**: `--surface` 배경, `--shadow-2`, `--brand` CTA

### 기본 정보 섹션
- **레시피명 입력**:
  - placeholder: "예: 김치찌개"
  - --surface-fill 배경, --radius-sm, 16px 패딩
  - text-base, --foreground
  - 최대 200자
- **기본 인분 stepper**:
  - 수평 레이아웃: [−] 숫자 [+]
  - 버튼: 44×44px 터치 타겟, --brand 색상
  - 최소값 1, 기본값 2
- **토큰**: `--surface-fill`, `--brand`, `--radius-sm`

### 재료 섹션
- **기본 상태**: 재료 카드 리스트 (세로 나열)
- **재료 카드**:
  - 좌측: 재료명 (text-base, --foreground)
  - 중앙: 수량/단위 또는 "약간" (text-sm, --text-2)
  - 우측: [×] 삭제 버튼 (44×44px)
  - --surface 배경, --line 테두리, --radius-md
  - 12px 수직 패딩, 16px 수평 패딩
- **재료 타입 구분**:
  - QUANT: 수량 + 단위 표시 (예: "200g", "2큰술")
  - TO_TASTE: "약간" 텍스트만 (--text-3 색상)
- **[+ 재료 추가] 버튼**:
  - --olive 색상, --radius-sm
  - 44px 높이, 전체 너비
  - 클릭 시 재료 입력 모달/바텀시트 열림
- **Empty**: 재료가 없으면 "재료를 추가해주세요" 안내 (--muted)
- **토큰**: `--surface`, `--line`, `--olive`, `--text-2/3`, `--radius-md`

### 조리 과정 섹션
- **기본 상태**: 스텝 카드 리스트 (세로 나열)
- **스텝 카드**:
  - 상단: 스텝 번호 + 조리방법 badge
    - 예: "1. 볶기" (조리방법 색상 배경, 흰색 텍스트)
    - --cook-{method} 색상 (design-tokens.md 기준)
  - 본문: 조리 설명 (text-base, --foreground)
  - 우측 상단: [×] 삭제 버튼 (44×44px)
  - --surface 배경, --line 테두리, --radius-md
  - 16px 패딩
- **[+ 조리 과정 추가] 버튼**:
  - --olive 색상, --radius-sm
  - 44px 높이, 전체 너비
  - 클릭 시 스텝 입력 모달/바텀시트 열림
- **Empty**: 스텝이 없으면 "조리 과정을 추가해주세요" 안내 (--muted)
- **토큰**: `--cook-*` 조리방법 색상, `--surface`, `--line`, `--olive`, `--radius-md`

- **세로 스크롤**: 전체 페이지 (기본 정보 → 재료 → 조리 과정)

## 디자인 결정 사항

### 1. 화면 형태: Full Page
- **이유**:
  - 직접 레시피 등록은 독립적인 맥락 전환과 긴 작성 흐름이 필요
  - 재료 추가, 스텝 추가 등 여러 단계의 입력이 필요
  - Modal/Sheet는 짧은 선택/필터에 적합하나, 다단계 작성 흐름에는 Full Page가 더 적합
- **출처**: `mobile-ux-rules.md` §6

### 2. 재료/스텝 추가: Modal/Bottom Sheet
- **이유**:
  - 재료 추가, 스텝 추가는 원래 보던 컨텍스트(입력 중인 레시피)를 유지해야 하는 작업
  - 짧은 선택 + 부가 입력 → Modal/Sheet 우선 검토
  - Full Page로 바꾸면 학습 비용 증가
- **출처**: `mobile-ux-rules.md` §6

### 3. 등록 완료 후 선택지 제공
- **이유**:
  - 사용자는 등록 후 "끼니에 추가" 또는 "레시피 상세로 이동" 중 선택 가능
  - "끼니에 추가"는 바로 MEAL_SCREEN 복귀 (빠른 경로)
  - "레시피 상세로 이동"은 플래너 추가/저장/좋아요 등 추가 작업 가능 (유연한 경로)
- **출처**: workpack README §Primary User Path

### 4. 조리방법 색상 시스템
- **이유**:
  - 조리방법별 색상은 COOK_MODE와 일관성 유지
  - 사용자가 조리방법을 시각적으로 빠르게 구분 가능
  - 조리방법 badge는 --cook-{method} 토큰 사용 (design-tokens.md 기준)
- **출처**: `design-tokens.md` 요리모드 조리방법 색상

### 5. Baemin vocabulary/material 사용
- **이유**:
  - h8 matrix에서 `prototype-derived design`으로 분류
  - Baemin vocabulary/material/tokens 사용하되, near-100% parity 타겟 아님
  - 기존 토큰 시스템(`--brand`, `--olive`, `--surface` 등)과 조화
- **출처**: h8 workpack, `BAEMIN_STYLE_DIRECTION.md`

### 6. H5 modal system redesign 적용
- **이유**:
  - 재료 추가 모달, 스텝 추가 모달은 H5 Quiet Kitchen Sheets 기준 적용
  - `olive base + thin orange highlight`
  - icon-only close (우측 상단)
  - eyebrow 제거
  - compact copy
- **출처**: 화면정의서 v1.5.0 → v1.5.1 변경 (H5 modal system redesign)

### 7. 작은 모바일 sentinel 고려
- **이유**:
  - 320px 수준에서도 레이아웃 붕괴 방지
  - CTA 잘림, 핵심 정보 줄바꿈 붕괴, 터치 타겟 축소 방지
  - 폰트, 여백, 카드 크기 비례 축소
- **출처**: `mobile-ux-rules.md` §5

### 8. Primary CTA: 우측 상단 [저장]
- **이유**:
  - 핵심 행동은 시각 위계상 가장 강해야 함
  - --brand 색상으로 구분
  - 필수 필드 미입력 시 disabled (--text-4)
- **출처**: `mobile-ux-rules.md` §3

### 9. 재료/스텝 Empty 상태 안내
- **이유**:
  - 사용자가 "지금 뭘 해야 하는지" 바로 이해 가능
  - Empty copy로 행동 유도 ("재료를 추가해주세요", "조리 과정을 추가해주세요")
- **출처**: `mobile-ux-rules.md` §3, 화면정의서 전역 규칙 §0-2

### 10. 로그인 게이트 + return-to-action
- **이유**:
  - 비로그인 상태에서 [저장] 시도 → 안내 모달 → 로그인 → 등록 폼 자동 복귀
  - 즉시 로그인 이동 X
  - 입력 내용 보존 (세션/로컬스토리지)
- **출처**: 화면정의서 전역 규칙 §0-3

## 상태별 UI 변형

### 1. Loading (조리방법 목록 로딩)
```
┌─────────────────────────────────┐
│  조리 과정 추가                  │
│                                 │
│  [스켈레톤 8개 칩]               │  ← 조리방법 로딩 중
│                                 │
└─────────────────────────────────┘
```
- **표시**: 조리방법 칩 영역에 스켈레톤 8개
- **조건**: GET /cooking-methods 응답 대기 중

### 2. Empty (신규 등록 상태)
```
┌─────────────────────────────────┐
│  🥬 재료                         │
│  재료를 추가해주세요              │  ← --muted, text-sm
│  [+ 재료 추가]                   │
│                                 │
│  👨‍🍳 조리 과정                   │
│  조리 과정을 추가해주세요         │  ← --muted, text-sm
│  [+ 조리 과정 추가]              │
└─────────────────────────────────┘
```
- **표시**: 섹션별 Empty 안내 + CTA 버튼
- **조건**: 재료 또는 스텝이 0개일 때

### 3. Error (Validation Error)
```
┌─────────────────────────────────┐
│  레시피명 (필수)                 │
│  ┌───────────────────────────┐  │
│  │ [빈 입력]                  │  │  ← --brand-deep border (error)
│  └───────────────────────────┘  │
│  레시피명을 입력해주세요          │  ← --brand 색상, text-sm
└─────────────────────────────────┘
```
- **표시**: 필드별 error 메시지 + 빨간 테두리
- **조건**: [저장] 시도 시 필수 필드 누락 또는 타입 제약 위반

### 4. Error (500 / Network Error)
```
┌─────────────────────────────────┐
│  ⚠️ 레시피 등록 실패              │  ← text-lg, --foreground
│                                 │
│  네트워크 오류가 발생했어요       │  ← text-base, --text-2
│  잠시 후 다시 시도해주세요        │
│                                 │
│  [다시 시도]                     │  ← --brand CTA
│  [닫기]                         │
└─────────────────────────────────┘
```
- **표시**: 오류 안내 모달 + [다시 시도] 버튼
- **조건**: POST /recipes 500 또는 네트워크 오류

### 5. Unauthorized (비로그인)
```
┌─────────────────────────────────┐
│  🔒 로그인이 필요해요             │  ← text-lg, --foreground
│                                 │
│  레시피를 등록하려면              │  ← text-base, --text-2
│  로그인이 필요해요                │
│                                 │
│  [로그인]                        │  ← --brand CTA
│  [취소]                         │
└─────────────────────────────────┘
```
- **표시**: 로그인 게이트 모달
- **조건**: 비로그인 상태에서 [저장] 시도
- **로그인 후**: 등록 폼 자동 복귀 (입력 내용 복원)

### 6. Read-only (등록 완료 후)
- **N/A**: 등록 완료 후에는 등록 완료 모달로 이동하며, 화면은 닫힘
- 등록된 레시피는 RECIPE_DETAIL에서 read-only로 확인 가능 (수정 불가)

## design-critic 검토 필요 항목

- [ ] 재료 추가 모달의 검색 결과 리스트가 너무 길면 스크롤 경계 명확히 표시
- [ ] 조리방법 칩이 8개 이상이면 wrap grid가 화면을 벗어나지 않는지 확인
- [ ] 320px 폭에서 [저장] 버튼 텍스트 잘림 여부 확인
- [ ] 재료/스텝 카드의 [×] 버튼 터치 타겟이 44×44px 이상인지 확인
- [ ] 키보드 열림 시 입력 필드가 가려지지 않는지 확인 (virtual keyboard inset)
- [ ] 등록 완료 모달의 [끼니에 추가] / [레시피 상세로 이동] 버튼 위계 명확한지 확인
- [ ] 조리방법 색상이 COOK_MODE와 일관성 있는지 확인 (--cook-* 토큰)
- [ ] Baemin vocabulary/material 사용이 기존 토큰 시스템과 조화를 이루는지 확인
- [ ] 로그인 게이트 후 return-to-action 흐름이 자연스러운지 확인 (입력 내용 복원)
- [ ] 재료 타입 선택 UI가 QUANT/TO_TASTE 구분을 명확히 하는지 확인
