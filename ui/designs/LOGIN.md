## 컴포넌트 상세

### 앱 로고 / 브랜드 일러스트

- **기본 상태**: 로고 텍스트 또는 SVG 일러스트. text-3xl weight-800 `--brand` 색상. 아래에 슬로건 text-sm `--muted`.
- **로고 텍스트 기준**: "집밥" 또는 서비스명. 실제 SVG 에셋 확정 전까지 텍스트 폴백 사용.
- **Loading**: 해당 없음 (정적 에셋).
- **Empty**: 해당 없음.
- **Error**: 이미지 로드 실패 시 텍스트 로고 폴백.

### 소셜 로그인 버튼 (3개)

- **기본 상태**: 버튼 3개 세로 배열. 각 버튼 width 100%, height 52px (터치 타겟 44px 초과 충족). border-radius 12px. 좌측 소셜 아이콘 24×24px, 중앙 텍스트.

| 버튼 | 배경 | 텍스트 색 | 테두리 |
|------|------|----------|--------|
| 카카오로 시작하기 | `#FEE500` (브랜드 예외) | `--foreground` | 없음 |
| 네이버로 시작하기 | `#03C75A` (브랜드 예외) | `#ffffff` | 없음 |
| Google로 시작하기 | `--surface` | `--foreground` | `--line` |

- **Loading (진행 중)**: 탭된 버튼에 스피너 표시. 나머지 버튼 opacity-60, pointer-events-none. 이중 탭 방지.
- **Empty**: 해당 없음.
- **Error**: 소셜 인증 실패 시 `authError=oauth_failed` URL 파라미터와 함께 LOGIN 화면으로 리다이렉트. 에러 배너 표시.

### 에러 배너 (`authError=oauth_failed`)

- **표시 조건**: URL에 `authError=oauth_failed` 파라미터 존재 시.
- **위치**: 로고 영역 위, 화면 상단.
- **스타일**: 배경 `rgba(255,108,60,0.08)` (--brand 연색), 테두리 left `--brand` 3px, 아이콘 ⚠, 텍스트 "로그인에 실패했어요. 다시 시도해주세요." text-sm `--foreground`.
- **해제**: 사용자가 소셜 버튼 재탭 시 자동 사라짐.

### 닉네임 입력 필드 (Step 2)

- **기본 상태**: placeholder "닉네임 입력", 우측 글자수 카운터 "0/30" text-xs `--muted`. 높이 52px. --surface bg, --line 테두리, radius 12px.
- **입력 중**: 테두리 `--brand` (포커스).
- **유효 상태 (2자 이상)**: 테두리 `--olive`. 하단 "좋은 닉네임이에요!" text-xs `--olive`.
- **에러 상태 (1자 이하)**: 하단 "2자 이상 입력해주세요" text-xs `--brand`. 테두리 `--brand`.
- **30자 도달**: 카운터 "30/30" `--brand` 색상. maxLength 차단으로 추가 입력 불가.
- **Loading**: 해당 없음 (클라이언트 유효성만).
- **Error (서버 422)**: Toast "닉네임 설정에 실패했어요. 다시 시도해주세요".

### 시작하기 버튼 (Step 2)

- **비활성 상태**: bg `--muted` opacity-30, text `--muted`, cursor-not-allowed. 탭 무반응.
- **활성 상태 (2자 이상)**: bg `--brand`, text white. 탭 가능.
- **Loading**: bg `--brand-deep`, 내부 스피너. pointer-events-none.

### 로그인 게이트 모달 (`LoginRequiredModal`)

- **기본 상태**: 하단에서 올라오는 바텀시트. dim overlay bg-black/50. 모달 시트 `--panel` bg, border-radius 20px (상단만).
- **Loading**: 해당 없음 (모달 표시 자체는 즉시).
- **Empty**: 해당 없음.
- **Error**: 해당 없음.

## 인터랙션 노트

| 액션 | 트리거 | 결과 | 로그인 필요 |
|------|--------|------|------------|
| 카카오 버튼 탭 | Step 1 소셜 버튼 탭 | OAuth 리다이렉트 → `POST /auth/login` | N (로그인 화면 자체) |
| 네이버 버튼 탭 | Step 1 소셜 버튼 탭 | OAuth 리다이렉트 → `POST /auth/login` | N |
| 구글 버튼 탭 | Step 1 소셜 버튼 탭 | OAuth 리다이렉트 → `POST /auth/login` | N |
| OAuth 성공 (신규) | `is_new_user=true` | Step 2(닉네임 설정)로 전환 | N |
| OAuth 성공 (기존) | `is_new_user=false` | `returnToAction` 확인 → 해당 화면 이동, 없으면 HOME | N |
| OAuth 실패 | 콜백 에러 | `authError=oauth_failed` 파라미터로 LOGIN 리다이렉트 → 에러 배너 | N |
| 닉네임 입력 | Step 2 인풋 포커스 | 키보드 올라옴, 글자수 카운터 업데이트, 유효성 실시간 판단 | N |
| 시작하기 탭 (유효) | Step 2 버튼 탭 | `PATCH /auth/profile` { nickname } → 성공 시 `returnToAction` 확인 → 이동 | N |
| 시작하기 탭 (비유효) | Step 2 버튼 탭 | 무반응 | N |
| [로그인하기] 탭 (게이트 모달) | LoginRequiredModal 버튼 탭 | `returnToAction` 파라미터 포함하여 LOGIN으로 이동 | N |
| [취소] 탭 (게이트 모달) | LoginRequiredModal 버튼 탭 | 모달 닫기, 이전 화면 유지 | N |
| 배경(dim) 탭 (게이트 모달) | 오버레이 탭 | 모달 닫기, 이전 화면 유지 | N |
| ESC 키 (게이트 모달) | 키보드 ESC | 모달 닫기, 이전 화면 유지 | N |
| 이용약관 링크 탭 | 하단 링크 탭 | 외부 URL (앱 내 웹뷰 또는 새 탭) | N |
| 개인정보처리방침 링크 탭 | 하단 링크 탭 | 외부 URL (앱 내 웹뷰 또는 새 탭) | N |

### return-to-action 대상 목록

| 원래 시도한 액션 | 복귀 후 자동 수행 |
|----------------|----------------|
| 좋아요 | 해당 레시피 좋아요 토글 |
| 저장 | 레시피북 선택 모달 오픈 |
| 플래너 추가 | 날짜/끼니 선택 모달 오픈 |
| 요리완료 저장 | 요리완료 처리 진행 |
| 플래너 탭 진입 | PLANNER_WEEK 진입 |
| 팬트리 탭 진입 | PANTRY 진입 |
| 마이페이지 탭 진입 | MYPAGE 진입 |

> `returnToAction` 파라미터는 1회 소비 후 URL에서 제거한다.
> 기존 회원(`is_new_user=false`)은 Step 2(닉네임 설정)를 표시하지 않는다.

### 로그인 게이트 진입 경로

- (A) 보호 액션(좋아요/저장/플래너 추가/요리완료 등) 시도 → `LoginRequiredModal` 팝업
- (B) 하단 탭에서 플래너/팬트리/마이페이지 직접 탭 (비로그인) → `LoginRequiredModal` 팝업
- (C) 보호 화면 URL 직접 접근 → `LoginRequiredModal` 팝업
- 공통: 모달 [로그인하기] → LOGIN 화면 (returnToAction 포함)

## 디자인 결정 사항

1. **하단 탭바 미표시**: LOGIN은 비로그인 전용 화면으로, 요구사항 §2-3에 따라 탭바를 노출하지 않는다. 로그인 완료 후 복귀하는 화면에서 탭바가 표시된다.

2. **게이트 모달 버튼 레이블 조정**: 화면정의서 §0-3은 "[로그인] [취소]", 유저플로우 v1.2는 "[로그인] [취소]", 세션1 와이어프레임은 "[로그인하기] [나중에]"로 표기가 다르다. 공식 문서(화면정의서, 요구사항) 기준으로 "[로그인하기] [취소]"로 통일한다. (세션 와이어프레임은 보조 참고이므로 공식 문서 우선 적용.)

3. **소셜 버튼 색상 예외**: 카카오 `#FEE500`, 네이버 `#03C75A`는 각 플랫폼 브랜드 가이드라인 준수를 위해 디자인 토큰 예외로 명시한다. 이 두 색상은 design-tokens.md에 등록하지 않고 컴포넌트 레벨에서 하드코딩으로 관리한다.

4. **구글 버튼 스타일**: 구글 버튼에 `--brand` 적용 여부 검토. 요청 컨텍스트에는 "구글 로그인 버튼 등 공통 CTA에 --brand 적용"이라고 명시됐으나, 구글 브랜드 가이드라인상 구글 버튼은 흰색 배경+테두리 표준이다. 구글 브랜드 가이드 준수를 우선하여 `--surface` 배경 + `--line` 테두리로 설계한다. [design-critic 검토 필요]

5. **에러 배너 위치**: `authError=oauth_failed`는 OAuth 콜백 실패 후 리다이렉트로 전달된다. 배너는 소셜 버튼 위(로고 아래)에 배치하여 흐름을 방해하지 않으면서 시각적으로 인지 가능하게 한다.

6. **Step 2 기존 회원 스킵 정책**: `is_new_user=false`인 기존 회원에게는 Step 2(닉네임 설정)를 절대 표시하지 않는다. 이는 세션 3 와이어프레임 비즈니스 로직 주의사항과 완전히 일치한다.

7. **신규 회원 자동 초기화**: `PATCH /auth/profile` 성공 시 서버가 끼니 컬럼 3개 + 레시피북 3개를 자동 생성한다. 클라이언트에서 별도 API 호출 불필요. UI 설계 범위 밖이나 UX 참고로 기록.

8. **이용약관/개인정보처리방침**: 화면정의서에 명시 없으나 세션 3 와이어프레임에 하단 링크로 포함되어 있다. 서비스 필수 요소이므로 포함한다. 충돌이 아닌 보완으로 판단.
