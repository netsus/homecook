## 화면 개요

MYPAGE 계정 섹션의 톱니바퀴(gear) 버튼에서 navigation push로 진입하는 설정 화면.
요리모드 화면 꺼짐 방지 토글, 닉네임 변경, 로그아웃, 회원탈퇴 기능을 제공한다.

- **권한**: 로그인 필수
- **API**: `PATCH /users/me/settings`, `PATCH /users/me`, `POST /auth/logout`, `DELETE /users/me`
- **진입**: MYPAGE 프로필 섹션의 gear 버튼
- **이탈**: 뒤로가기 -> MYPAGE / 로그아웃 -> HOME / 회원탈퇴 -> HOME

|------|--------|------|-----|------------|

### 토글 동작 상세

- **optimistic update**: 토글 탭 즉시 UI 변경 -> 백그라운드 API 호출
- 성공: 변경 유지
- 실패: 원래 값으로 롤백 + 에러 토스트
- 연속 탭 방지: 진행 중인 API가 있으면 debounce (300ms)
- Screen Wake Lock API (`navigator.wakeLock`)가 지원되지 않는 환경에서는:
  - 토글은 표시하되 탭 시 토스트 "이 기기에서는 지원되지 않아요"
  - 또는 서버에 설정값은 저장하되 클라이언트 측 미적용

## 320px 대응 (작은 모바일 sentinel)

- **앱바**: "설정" 제목 + 뒤로가기 버튼은 320px에서도 넉넉함
- **토글 항목**: 텍스트 영역이 좁아져도 제목/설명이 2행으로 자연스럽게 wrap. 토글 스위치는 우측 고정
- **닉네임 항목**: "닉네임" 라벨 + 닉네임 텍스트가 좁아져도 세로 2행이므로 안전. 긴 닉네임은 말줄임
- **로그아웃/회원탈퇴**: 중앙 정렬 텍스트이므로 폭 무관
- **닉네임 바텀시트**: 좌우 여백 --space-4 -> --space-3 (12px) 축소
- **컨테이너 좌우 여백**: --space-4 (16px) 유지 (항목 수 적어 축소 불필요)

320px에서 확인해야 할 blocker:
- [ ] 토글 제목 + 스위치가 한 행에 겹치지 않는지 (제목 2행 wrap 허용)
- [ ] 닉네임이 30자일 때 말줄임이 적용되고 chevron과 겹치지 않는지
- [ ] 바텀시트 입력 필드 + CTA 버튼이 키보드 위에서 가려지지 않는지
- [ ] 모든 버튼/터치 타겟이 44px 이상인지

## 디자인 결정 사항

1. **별도 화면 채택 (바텀시트 대신)**: 화면정의서에서 "별도 화면 또는 바텀시트"로 형태가 열려 있었다. 설정 항목이 4개(토글 + 닉네임 + 로그아웃 + 회원탈퇴)이고, 닉네임 변경 시 하위 바텀시트를 추가로 열어야 하므로 바텀시트 안에 바텀시트를 중첩하는 것보다 별도 화면이 자연스럽다. navigation push 구조를 채택하여 MYPAGE에서 뒤로가기로 복귀한다.

2. **하단 탭바 미표시**: SETTINGS는 하단 탭 IA(홈/플래너/팬트리/마이)의 직접 하위가 아니라 MYPAGE에서 push로 진입하는 화면이다. 일반적인 모바일 앱 패턴에 따라 push된 설정 화면에서는 탭바를 숨기고, 앱바에 뒤로가기 버튼을 제공한다.

3. **위험 행동 시각적 분리**: 로그아웃은 --text-2 텍스트의 중립 카드 버튼으로, 회원탈퇴는 --text-3 밑줄 텍스트 버튼으로 무게를 낮췄다. 회원탈퇴는 시각적으로 가장 약하게 표현하되, 확인 다이얼로그에서 --danger 색상으로 위험성을 강조한다. 이는 실수로 누르는 것을 방지하면서도 필요 시 접근 가능하게 한다.

4. **확인 다이얼로그 필수**: 로그아웃과 회원탈퇴 모두 되돌릴 수 없는 행동이므로 확인 다이얼로그를 거친다. 로그아웃은 --brand 버튼(중립적 확인), 회원탈퇴는 --danger 버튼(위험 경고)으로 시각적 긴장감을 다르게 설정했다.

5. **닉네임 변경 바텀시트 패턴**: Quiet Kitchen Sheets 규칙(eyebrow 없음, icon-only close)을 따른다. 닉네임 변경은 짧은 입력이므로 바텀시트가 적합하다(mobile-ux-rules SS 6). 키보드가 올라올 때 시트 콘텐츠가 밀려 올라가도록 구현한다.

6. **토글 optimistic update**: 화면 꺼짐 방지 토글은 설정 변경의 즉시 반응성이 중요하므로 optimistic update 패턴을 사용한다. API 실패 시 롤백하고 에러를 안내한다.

7. **섹션 분리**: "앱 설정"과 "계정 관리" 두 섹션으로 분리하여 정보 위계를 명확히 했다. 앱 설정은 기능 토글, 계정 관리는 사용자 정보와 계정 행동을 담당한다.

8. **Screen Wake Lock API 비지원 대응**: 모든 브라우저에서 Screen Wake Lock API를 지원하지 않으므로, 비지원 환경에서의 fallback 정책을 포함했다. 서버에 설정값은 저장하되 클라이언트 적용은 환경에 따른다.

## Anchor Screen 영향 분석

- **SETTINGS는 anchor screen이 아니다** (anchor-screens.md 확인).
- **Anchor extension 해당 여부**: 해당하지 않음. 기존 anchor screen(HOME, RECIPE_DETAIL, PLANNER_WEEK)에 영향 없음.
- **모바일 UX 리스크**: 낮음. 항목 수가 적고, 패턴이 일반적인 모바일 설정 화면과 동일하다. 닉네임 변경 바텀시트에서 키보드-시트 상호작용만 주의하면 된다.

## 접근성 노트

- **앱바 뒤로가기**: `aria-label="뒤로가기"`, `role="button"`
- **토글 스위치**: `role="switch"`, `aria-checked="true/false"`, `aria-label="요리모드 화면 꺼짐 방지"`
- **닉네임 항목**: `role="button"`, `aria-label="닉네임 변경, 현재 닉네임: 집밥러"`
- **닉네임 바텀시트**: `role="dialog"`, `aria-modal="true"`, 열릴 때 입력 필드에 포커스
- **닉네임 입력 필드**: `aria-describedby` 도움말/에러 메시지 연결
- **로그아웃 확인 다이얼로그**: `role="alertdialog"`, `aria-modal="true"`, 포커스 트랩
- **회원탈퇴 확인 다이얼로그**: `role="alertdialog"`, `aria-modal="true"`, 포커스 트랩
- **색상 대비**: --text-2 (#495057) on --surface (#ffffff) = 6.6:1 (AA 통과), --text-3 (#868E96) on --surface = 3.7:1 (AA large text 통과)
- **터치 타겟**: 모든 인터랙티브 요소 최소 44x44px

## design-critic 검토 필요 항목

- [ ] 로그아웃 버튼이 카드 스타일(--surface, radius)로 충분히 "탭 가능" 어포던스를 전달하는지
- [ ] 회원탈퇴 텍스트 버튼이 --text-3 + 밑줄로 너무 약하지 않은지 (발견성 vs 실수 방지 균형)
- [ ] 닉네임 변경 바텀시트에서 키보드가 올라올 때 CTA 버튼이 가려지지 않는지
- [ ] 토글 설명 텍스트("요리 중 화면이 꺼지지 않아요")가 기능을 충분히 설명하는지
- [ ] 로그아웃 후 HOME 이동 vs LOGIN 이동 — 어느 쪽이 더 자연스러운지 (현재: HOME)
- [ ] 회원탈퇴 확인 다이얼로그에 "탈퇴" 텍스트 재입력 같은 추가 확인 단계가 필요한지
- [ ] Screen Wake Lock API 비지원 시 토글 숨김 vs 비활성 표시 — 어느 쪽이 혼란이 적은지
- [ ] 설정 항목이 4개뿐인 현재 구조에서 세로 여백이 과도하게 넓어 보이지 않는지
