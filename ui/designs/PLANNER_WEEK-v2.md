## 핵심 변경 요약

| 항목 | v1 (현행 2×2 grid) | v2 (day-card slot rows) |
|------|-------------------|------------------------|
| 카드 본문 | 2×2 grid (끼니가 가로로 2열씩 배치) | 세로 slot row (끼니 1개 = 1행) |
| 가로 스크롤 | planner 내부 scroller (localized) | **없음** (v1.5.1에서 planner-level 잠금 supersede) |
| 슬롯당 유효 너비 | viewport ÷ 4 ≈ 80px | 카드 너비 전체 ≈ 350px |
| 식사명 표시 | 짧은 truncate 필수 | 1행 자연 wrap 가능 |
| 390px 첫 화면 노출 | ~2일 (행 높이 압축 필요) | ~2.5일 자연 노출 |
| 320px 안정성 | 슬롯 폭 축소 위험 | 가로 제약 없음 |

## 핵심 컴포넌트 명세

### Day Card 구조

```
┌──────────────────────────────────────────┐
│ [요일 배지]  M월 D일                     │  card header
├──────────────────────────────────────────┤
│ 아침   [식사명 또는 empty pill]  [chips] │  slot row
│ 점심   [식사명 또는 empty pill]  [chips] │
│ 간식   [식사명 또는 empty pill]  [chips] │
│ 저녁   [식사명 또는 empty pill]  [chips] │
└──────────────────────────────────────────┘
```

### Slot Row 레이아웃 (1행)

```
[끼니명 4ch]  [식사명 flex-1, 1행 truncate]  →  [인분 chip] [상태 chip]
```

- **끼니명**: `아침 / 점심 / 간식 / 저녁` — 생략 금지, text-xs, muted color
- **식사명**: flex-1, 1행 truncate, 전체 row가 tap target
- **빈 슬롯**: `─ 비어 있음 ─` — text-xs, muted, separator 스타일
- **인분 chip**: `N인분`, text-xs, secondary variant
- **상태 chip**: 상태별 색상 (아래 참조)
- **터치 타겟**: 각 slot row 최소 height 44px

### 상태 Chip 색상

| status | label | 방향 |
|--------|-------|------|
| `registered` | 등록 | neutral / muted |
| `shopping_done` | 장보기 | primary (브랜드 포인트) |
| `cook_done` | 완료 | success / green |

## 5-Column 대응

5번째 끼니(예: 야식) 추가 시 slot row 1행을 추가한다.
카드 높이만 증가 — 가로 폭과 가로 밀도에 영향 없음.

```
│ 아침   [식사명]              [chips] │
│ 점심   [식사명]              [chips] │
│ 간식   ─ 비어 있음 ─                │
│ 저녁   [식사명]              [chips] │
│ 야식   ─ 비어 있음 ─                │  ← 행 1개 추가
```

**5-column 정보 축약 원칙**:
1. 끼니명 생략 금지
2. 식사명 말줄임(`…`) 1행까지 허용
3. serving chip: 공간 부족 시 생략 가능
4. status chip: 우선 유지
5. `5 equal-width slots in one row` 방식 채택 금지
