# PLANNER_WEEK v2 — 모바일 day-card 리디자인 브리프

> 기준 문서: 화면정의서 v1.2.1 §5 / 요구사항기준선 v1.6.1 §1-4 / API v1.2.1 §3
> 참고 입력: 팀장님 제공 모바일 플래너 레퍼런스 이미지 (2026-04-10)
> 목적: slice06 전 `PLANNER_WEEK` 모바일 baseline 재정의

---

## 왜 바꾸나

현재 `PLANNER_WEEK` baseline은 계약과 CRUD는 닫았지만,
모바일에서 아래 문제가 남아 있다.

- 첫 화면에 실제 식단보다 설명/칩/여백이 먼저 보인다.
- 주간 범위 이동 컨트롤이 planner 본문과 멀다.
- 같은 날짜의 식단을 한 번에 읽기보다 `날짜 x 끼니 표`를 스캔해야 한다.
- localized horizontal scroll이 남아 있어 "한눈에 보는 플래너" 감각이 약하다.
- 4~5컬럼이 되었을 때 모바일 가독성 전략이 불분명하다.

v2의 목표는 "표 구조의 정보 정합성"을 유지하면서도,
모바일에서는 `하루 1카드 + 세로 스크롤 중심`으로 더 빠르게 읽히는 planner를 만드는 것이다.

---

## 핵심 방향

- 모바일 기본형은 `가로 스크롤 없는 주간 overview`를 우선한다.
- 같은 날짜의 끼니는 한 day card 안에서 함께 읽힌다.
- 주간 범위와 주간 이동 컨트롤은 planner 바로 위에 붙는다.
- 긴 설명 문장보다 실제 day card를 첫 화면에 더 많이 노출한다.
- 4~5컬럼 지원은 유지하되, 한 슬롯이 작아지면 `보조 row / disclosure / sheet`로 전환한다.

## 레이아웃 와이어프레임

### 기본 화면

```text
┌─────────────────────────────────────┐
│ 다음주에요   [이번주로 가기]        │
│ 4월 12일 ~ 4월 18일                │
│ [이전 주]                    [다음 주]│
├─────────────────────────────────────┤
│ 일 12  월 13  화 14  수 15  목 16   │
│ 금 17  토 18                        │
├─────────────────────────────────────┤
│ 6일 기록, 총 12끼니                 │
├─────────────────────────────────────┤
│ [일] 4월 12일                  [⋯] │
│ 아침      점심      저녁            │
│ 김치찌개   샐러드    빈 상태         │
│ 2인분      1인분                    │
│ 등록완료   장보기완료               │
├─────────────────────────────────────┤
│ [월] 4월 13일                  [⋯] │
│ 아침      점심      저녁            │
│ ... 같은 card 패턴 반복 ...         │
└─────────────────────────────────────┘
```

### Empty 상태

```text
┌─────────────────────────────────────┐
│ 이번 주 4월 12일 ~ 4월 18일         │
│ [이전 주]                    [다음 주]│
├─────────────────────────────────────┤
│ 아직 등록된 식사가 없어요            │
│ 오늘부터 채울 수 있게 끼니만 빠르게   │
│ 정리해둘까요?                  [관리] │
├─────────────────────────────────────┤
│ [일] 4월 12일                       │
│ 아침      점심      저녁            │
│ 비어 있음  비어 있음  비어 있음      │
└─────────────────────────────────────┘
```

### 4~5컬럼 상태

```text
┌─────────────────────────────────────┐
│ [화] 4월 14일                  [⋯] │
│ 아침      점심      저녁            │
│ ... primary 3 slots ...             │
│ [간식 있음] [야식 비어 있음]        │  ← secondary row or disclosure
└─────────────────────────────────────┘
```

핵심은 `5 equal-width slots in one row`를 모바일 기본안으로 쓰지 않는 것이다.

## 핵심 컴포넌트

### 1. Week Context Bar

- planner 본문 바로 위에 둔다.
- 현재 보고 있는 범위와 현재 주 여부를 함께 보여준다.
- 예:
  - `이번주`
  - `다음주에요 + 이번주로 가기`
  - `지난주에요 + 이번주로 가기`
- 주간 이동 컨트롤은 같은 블록 안에 둔다.

### 2. Weekday Strip

- 요일과 날짜를 매우 압축된 한 줄 요약으로 보여준다.
- 현재 범위의 7일을 빠르게 스캔하는 역할이다.
- 스크롤 가능한 탭이 아니라 `현재 주 미리보기`에 가깝다.

### 3. Day Card

- 모바일 planner의 기본 단위다.
- 같은 날짜 식단을 한 card 안에 모은다.
- card header에는 최소한 아래만 둔다.
  - 요일 배지
  - 날짜
  - 보조 액션 진입 affordance (`⋯` 또는 icon)
- 카드 간격은 현재보다 촘촘하게 가져간다.

### 4. Meal Slots Inside Day Card

- 기본 3슬롯은 한 row에서 함께 보인다.
- 각 슬롯은 아래 우선순위로 보인다.
  - 끼니명
  - 식사명 또는 empty text
  - 인분/상태
- empty text는 현재보다 짧고 반복 피로가 적어야 한다.
  - 예: `비어 있음`, `추가 전`

### 5. Column Management Entry

- `컬럼 추가 / 이름 수정 / 삭제 / reorder`는 planner의 핵심 overview보다 한 단계 뒤로 뺀다.
- 상시 거대한 헤더 편집 UI로 노출하지 않는다.
- 권장 방식:
  - planner toolbar의 `끼니 관리` 버튼
  - bottom sheet 또는 관리 전용 panel
- 단, drag handle 기반 reorder 계약은 유지한다.

## 5컬럼 처리 전략

## 문제 정의

카드 하나 안에 3슬롯은 안정적이지만,
4~5개를 같은 줄에 넣으면 각 슬롯이 너무 좁아져 아래 문제가 생긴다.

- 식사명 읽기 어려움
- 상태 뱃지와 인분 정보가 눌림
- 빈 상태 문구 반복이 더 지저분해짐
- day card가 "한눈에 보는 카드"가 아니라 "작은 표"처럼 다시 느껴짐

## 추천안

### 기본 원칙

- 모바일 day card의 `primary visible slots`는 최대 3개다.
- 정렬 우선순위는 현재 `sort_order`를 그대로 따른다.
- 4번째, 5번째 컬럼은 보조 정보로 취급한다.

### 3컬럼

- `1 row x 3 slots`
- 가장 이상적인 기본형

### 4컬럼

- 1안: `primary 3 + secondary 1`
- day card 하단에 4번째 컬럼을 compact secondary row로 노출
- 4번째 컬럼에 식사가 없으면 chip 수준으로 더 축약 가능

### 5컬럼

- 권장: `primary 3 + 추가 끼니 2개 disclosure`
- card 하단에 `추가 끼니 2개`, `간식 1 / 야식 0` 같은 summary를 노출
- 사용자가 열면:
  - inline secondary row 2칸
  - 또는 bottom sheet

## 비권장안

- 모바일에서 4개/5개를 동일 폭 한 줄로 배치
- 5개를 보여주기 위해 각 슬롯의 타이포와 여백을 과도하게 줄이기
- 4번째/5번째 끼니 때문에 전체 day card 높이가 매번 크게 출렁이는 구조

## 향후 확장 옵션

- 사용자가 `상위 3개 primary 끼니`를 pin하는 기능
- `추가 끼니`를 열었는지 기억하는 per-user preference
- 월간 보기에서는 다른 density 전략 사용

---

## 상호작용 규칙

- 주간 범위 이동은 planner 상단 bar에서 바로 수행한다.
- 세로 스크롤은 day card 목록 탐색용이다.
- 주간 이동 gesture는 나중에 넣더라도, 명시적 버튼이 항상 우선이다.
- 같은 날짜의 meal slot 탭은 이후 slice에서 `MEAL_SCREEN` 진입으로 연결한다.
- day card 우측 보조 액션은 추후:
  - day-level quick add
  - 끼니 관리 진입
  - 추가 끼니 펼치기
  중 하나로 쓸 수 있지만, slice06 전에는 과도하게 열지 않는다.

---

## 정보 밀도 기준

- standard mobile viewport에서 첫 화면에 `2일 이상`의 day card가 일부라도 보여야 한다.
- 긴 설명 문단은 최대 1문장으로 제한한다.
- 주간 범위 bar와 첫 day card 사이에는 별도 통계 카드나 큰 요약 카드가 끼지 않는다.
- 날짜칸과 끼니칸의 vertical padding은 현재 baseline보다 더 압축한다.
- 같은 날짜의 empty state는 3칸에 장문 복붙 대신 짧은 상태 표현을 쓴다.

---

## current baseline 대비 주요 변경

- giant grid / localized horizontal scroll 중심에서 `세로 스크롤 day card` 중심으로 이동
- 상단 hero 설명 축소
- 주간 범위 이동 컨트롤을 planner 바로 위로 이동
- 날짜 단위 overview 강화
- 컬럼 관리 UI를 상시 노출 대신 2차 진입 구조로 이동
- 5컬럼 대응을 "폭 축소"가 아니라 "정보 우선순위 재배치"로 처리

## 구현 전 메모

- 이 문서는 `slice06` 전에 planner mobile baseline을 다시 잠그기 위한 목표안이다.
- 현재 구현이 이 문서와 다를 수 있다.
- 구현 착수 전 authority preflight에서 아래를 확인한다.
  - first viewport에 실제 day card가 충분히 보이는가
  - 주간 컨트롤이 planner 본문과 붙어 있는가
  - 같은 날짜 식단이 한 card 안에서 읽히는가
  - 4~5컬럼 처리 방식이 가독성을 해치지 않는가
