## Option A — Table/Grid Model (현행 baseline)

```text
┌─────────────────────────────────────────────┐
│ HOMECOOK                                    │ ← 공통 브랜드 헤더
├─────────────────────────────────────────────┤
│ [장보기]  [요리하기]  [남은요리]             │ ← compact secondary toolbar
├─────────────────────────────────────────────┤
│ 4월 14일 ~ 4월 20일          [이번주로 가기] │ ← week context bar
│ ← [화 14  수 15  목 16  금 17  토 18] →    │ ← weekday strip (swipe)
├─────────────────────────────────────────────┤
│         아침    점심    간식    저녁          │ ← 끼니 헤더 행
├─────────────────────────────────────────────┤
│ 화 14  │김치찌개│ 샐러드 │        │        │
│        │ 2인·등 │ 1인·장 │ 비어있 │ 비어있 │
├─────────────────────────────────────────────┤
│ 수 15  │        │        │        │        │
│        │ 비어있 │ 비어있 │ 비어있 │ 비어있 │
├─────────────────────────────────────────────┤
│ 목 16  │된장찌개│        │        │        │
│        │ 2인·등 │ 비어있 │ 비어있 │ 비어있 │
├─────────────────────────────────────────────┤
│ ...                                         │
└─────────────────────────────────────────────┘
```

**특성**:
- 가로 축 = 끼니(4열), 세로 축 = 날짜
- 끼니별 세로 스캔 가능 ("이번 주 아침에 뭘 먹는지")
- 4열이 모바일 폭 안에 들어가야 하므로 슬롯당 너비 약 80~90px (390px 기준)
- 5컬럼 대응 시 슬롯당 너비 약 65px → 밀도 붕괴 위험

**현재 구현 상태**: slice05/06 authority `pass`  
(`2×2 meal slot grid`, scroll containment localized, M2/M3 polish 완료)

## Option B — Day-Card Model (전환 방향)

```text
┌─────────────────────────────────────────────┐
│ HOMECOOK                                    │ ← 공통 브랜드 헤더
├─────────────────────────────────────────────┤
│ [장보기]  [요리하기]  [남은요리]             │ ← compact secondary toolbar
├─────────────────────────────────────────────┤
│ 4월 14일 ~ 4월 20일          [이번주로 가기] │ ← week context bar
│ ← [화 14  수 15  목 16  금 17  토 18] →    │ ← weekday strip (swipe)
├─────────────────────────────────────────────┤
│ ┌───────────────────────────────────────┐   │
│ │ 화요일 4월 14일                       │   │ ← day card
│ │                                       │   │
│ │ 아침   김치찌개              2인·등록 │   │
│ │ 점심   샐러드                1인·장보 │   │
│ │ 간식   비어 있음                      │   │
│ │ 저녁   비어 있음                      │   │
│ └───────────────────────────────────────┘   │
│                                             │
│ ┌───────────────────────────────────────┐   │
│ │ 수요일 4월 15일                       │   │ ← day card
│ │                                       │   │
│ │ 아침   비어 있음                      │   │
│ │ 점심   비어 있음                      │   │
│ │ 간식   비어 있음                      │   │
│ │ 저녁   비어 있음                      │   │
│ └───────────────────────────────────────┘   │
│                                             │
│ ┌───────────────────────────────────────┐   │
│ │ 목요일 4월 16일                       │   │ ← day card (partial visible)
│ │ 아침   된장찌개              2인·등록 │   │
│ │ ...                                   │   │
└─────────────────────────────────────────────┘
```

**특성**:
- 날짜 단위 카드 → 카드 경계로 "하루 한 덩어리" 인지 강화
- 각 슬롯 행 = 끼니명 + 식사명(or 빈 상태) + 상태 chip
- 세로 스크롤만으로 탐색 (page-level horizontal scroll 없음)
- 슬롯당 너비 제약이 사라짐 — 식사명 텍스트가 더 넓게 읽힘
- 390px 기준 day card 1개 높이 ≈ 140~160px → 첫 화면에 약 2.5~3일 노출 가능

## 5-Column 대응 비교

### Table/Grid에서 5-column

```text
│         아침  점심  간식  저녁  야식  │
│ 화 14  │ ... │ ... │ ... │ ... │ ... │
```
- 슬롯당 너비 ≈ 65px (390px, 5열)
- 식사명 잘림, chip 생략 불가피
- 밀도 붕괴 위험 높음

### Day-Card에서 5-column

```text
│ 화요일 4월 14일                       │
│ 아침   김치찌개              2인·등록 │
│ 점심   샐러드                1인·장보 │
│ 간식   비어 있음                      │
│ 저녁   비어 있음                      │
│ 야식   비어 있음                      │  ← 행 하나 추가
```
- 슬롯 1개 행 추가 → card 높이만 증가
- 가로 밀도 영향 없음
- 정보 축약 원칙 적용 용이 (§ 5-column 축약 원칙 참조)

## 결정 요약

| 항목 | Table/Grid | Day-Card | 승자 |
|------|-----------|----------|------|
| "하루 식단 한 덩어리" 인지 | 약함 (행 단위) | 강함 (카드 단위) | Day-Card |
| Mobile 2일+ overview (390px) | 행 높이 공격적 압축 필요 | 자연스럽게 2.5일 이상 가능 | Day-Card |
| 끼니 축 스캔 ("이번 주 아침") | 가능 (열 방향) | 어려움 | Table/Grid |
| 5-column 대응 | 밀도 붕괴 위험 | 행 추가로 격리 | Day-Card |
| Narrow (320px) 안정성 | 슬롯 폭 축소 위험 | 가로 제약 없음 | Day-Card |
| 현행 구현 자산 재사용 | 전부 재사용 | 전면 교체 필요 | Table/Grid |
| Authority baseline 유지 | 유지 | 재설정 필요 | Table/Grid |

## 5-Column 정보 축약 원칙

Day-Card에서 5끼 column 대응 시:

1. **끼니명 생략 금지** — 아침/점심/간식/저녁/야식은 항상 노출
2. **식사명 말줄임은 1행까지** — `김치찌개 볶음밥...` 형태로 truncate
3. **chip 우선순위** — status chip 우선 유지, serving chip은 공간 부족 시 생략 가능
4. **빈 슬롯 pill** — `비어 있음` 1행 유지, 별도 설명 없음
5. **5 equal-width slots in one row는 기본안으로 채택하지 않음**
