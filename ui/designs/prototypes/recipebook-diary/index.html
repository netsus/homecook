          오늘의 기록

            책 느낌은 표지와 페이지에 집중

            레시피북 목록은 책 표지로 감정을 만들고, 상세에서는 펼친 페이지 레이아웃만 사용합니다. 실제 조작은 탭, 검색, 목록으로 익숙하게 유지합니다.

            모바일은 두 단계로 단순화

            앱에서는 책장 화면과 책 상세 화면을 나눴습니다. 상세 안에서는 상단에 목차를 먼저 보여주고, 선택한 레시피 내용을 바로 아래에서 읽게 해 전체 흐름을 단순하게 유지합니다.

            실서비스 연결은 기존 계약 유지

            기존 레시피북 API의 책 목록, 레시피 목록, 레시피 상세 이동을 그대로 소비할 수 있게 설계했습니다. 웹 책장의 이름 변경과 삭제는 실제 구현에서 모달과 API로 연결하면 됩니다.
