## 화면 개요

하단 탭바 [마이] 탭에서 진입하는 마이페이지 화면.
상단에 사용자 프로필(닉네임, 프로필 이미지, 소셜 제공자, 설정 버튼)이 표시되고,
아래 탭 영역에서 **레시피북**(기본 탭)과 **장보기 기록** 탭을 전환한다.

- **권한**: 로그인 필수
- **API**: `GET /users/me`, `GET /recipe-books`, `POST /recipe-books`, `PATCH /recipe-books/{book_id}`, `DELETE /recipe-books/{book_id}`, `GET /shopping/lists`
- **진입**: 하단 탭바 [마이]
- **이탈**: 하단 탭 전환 / ⚙️ → SETTINGS (17c) / 레시피북 항목 탭 → RECIPEBOOK_DETAIL (17b) / 장보기 기록 항목 탭 → SHOPPING_DETAIL (기존)

### Unauthorized 상태

비로그인 사용자가 하단 탭 [마이]를 탭한 경우.

```
+---------------------------------------+
|        로그인이 필요해요              |  <- 모달 오버레이
|                                       |     --panel 배경
|  마이페이지를 보려면                  |     --radius-xl 상단
|  로그인이 필요해요.                   |
|                                       |
|    [로그인]        [취소]             |  <- [로그인]: --brand filled
|                                       |     [취소]: --text-2, ghost
+---------------------------------------+

로그인 성공 후: return-to-action -> MYPAGE 자동 진입
```

### Read-only 상태

해당 없음. MYPAGE 자체는 항상 편집 가능하다.
장보기 기록 탭의 각 항목은 읽기 전용이지만, 탭 → `SHOPPING_DETAIL` (read-only mode)로 위임하므로
이 화면 자체에 read-only 모드는 존재하지 않는다.

## 컴포넌트 상세

### ProfileSection (프로필 섹션)

- **높이**: 약 80px (콘텐츠 기반)
- **프로필 이미지**: 48x48px 원형, --radius-full, fallback: 이니셜 아바타(--brand-soft 배경, --brand 텍스트)
- **닉네임**: text-lg, 700, --foreground
- **소셜 제공자**: text-sm, 400, --text-3 (예: "카카오 로그인")
- **⚙️ 버튼**: 우측 상단, 44x44px 터치 타겟
  - 아이콘: 톱니바퀴, 24px, --text-2
  - 탭 시: SETTINGS 이동 (17c에서 구현, 이 슬라이스에서는 route만 준비)
- **배경**: --surface
- **하단 구분**: --line (1px)
- **토큰 힌트**:
  - 배경 <- `--surface`
  - 닉네임 <- `--foreground`
  - 소셜 텍스트 <- `--text-3`
  - 이미지 테두리 <- `--line`
  - ⚙️ 아이콘 <- `--text-2`

### TabBar (탭 전환)

- **탭 수**: 2개 (레시피북 / 장보기 기록)
- **높이**: 44px
- **활성 탭**:
  - 텍스트: --brand (#ED7470), text-sm, 700
  - 밑줄: --brand, 2px, 탭 폭 기준
- **비활성 탭**:
  - 텍스트: --text-3 (#868E96), text-sm, 500
- **배경**: --surface
- **하단 경계**: --line (1px)
- **최소 터치 타겟**: 탭 전체 영역 44px 높이
- **토큰 힌트**:
  - 배경 <- `--surface`
  - 활성 텍스트/밑줄 <- `--brand`
  - 비활성 텍스트 <- `--text-3`
  - 하단 보더 <- `--line`

### SystemBookCard (시스템 레시피북 카드)

- **기본 상태**:
  - 아이콘 (24px) + 레시피북 이름 (text-base, 600, --foreground) + recipe_count (우측, text-sm, --text-3)
  - 가로 레이아웃: 아이콘 | 이름 | flexbox spacer | count
- **아이콘별 색상**:
  - my_added: --text-2 (#495057)
  - saved: --olive (#1f6b52)
  - liked: --brand (#ED7470)
- **카드 스타일**:
  - 배경: --surface (#ffffff)
  - border-radius: --radius-lg (16px)
  - box-shadow: --shadow-1
  - 패딩: --space-4 (16px) 수평, --space-3 (12px) 수직
  - 카드 간 간격: --space-2 (8px)
- **전체 영역 탭 가능**: 카드 전체가 터치 타겟 -> RECIPEBOOK_DETAIL (17b에서 구현, 이 슬라이스에서는 route만 준비)
- **최소 터치 높이**: 48px
- **토큰 힌트**:
  - 배경 <- `--surface`
  - 그림자 <- `--shadow-1`
  - 모서리 <- `--radius-lg`
  - 이름 <- `--foreground`
  - count <- `--text-3`

### CustomBookCard (커스텀 레시피북 카드)

- **기본 상태**:
  - 레시피북 이름 (text-base, 600, --foreground) + recipe_count (text-sm, --text-3)
  - ⋯ 메뉴 버튼: 우측, 44x44px 터치 타겟
- **⋯ 메뉴 탭 시**: 드롭다운 [이름 변경 / 삭제]
- **카드 스타일**: SystemBookCard와 동일 (배경, radius, shadow, 패딩)
- **전체 영역 탭 가능**: 카드 전체(⋯ 제외) -> RECIPEBOOK_DETAIL
- **토큰 힌트**: SystemBookCard와 동일 + ⋯ 아이콘 <- `--text-3`

### CreateBookCTA ([+ 새 레시피북] 카드)

- **스타일**: --surface 배경, --radius-lg
- **테두리**: 2px 점선 (dashed), --line
- **텍스트**: "+ 새 레시피북", text-base, 600, --brand
- **정렬**: 가운데 정렬
- **최소 높이**: 48px
- **터치 타겟**: 카드 전체
- **hover/pressed**: 배경 --brand-soft (#FDEBEA)
- **토큰 힌트**:
  - 배경 <- `--surface`, hover <- `--brand-soft`
  - 테두리 <- `--line` (dashed)
  - 텍스트 <- `--brand`

### ShoppingHistoryCard (장보기 기록 카드)

- **기본 상태**:
  - 제목: title (text-base, 600, --foreground)
  - 메타: "날짜범위 · 항목수" (text-sm, 400, --text-3)
  - 완료 뱃지 or 진행 중 뱃지
- **뱃지 스타일**:
  - 완료: --olive 텍스트, `rgba(31,107,82,0.1)` 배경, text-xs, --radius-full
  - 진행 중: --brand 텍스트, --brand-soft (#FDEBEA) 배경, text-xs, --radius-full
- **카드 스타일**: 동일 (--surface, --radius-lg, --shadow-1, --space-4 패딩)
- **전체 영역 탭 가능**: 카드 전체 -> SHOPPING_DETAIL (read-only mode)
- **카드 간 간격**: --space-2 (8px)
- **토큰 힌트**:
  - 배경 <- `--surface`
  - 그림자 <- `--shadow-1`

## 320px 대응 (작은 모바일 sentinel)

- 프로필 이미지: 48px 유지, 닉네임 1줄 말줄임 (충분한 공간)
- 탭 바: 두 탭이 50/50 분할, 텍스트 크기 유지
- 레시피북 카드: 좌우 패딩 --space-4 (16px) → --space-3 (12px) 축소
- 장보기 기록 카드: 동일 축소
- 컨테이너 좌우 여백: --space-4 → --space-3
- ⋯ 드롭다운: 카드 우측에 정렬, overflow 방지 (뷰포트 경계 체크)

320px에서 확인해야 할 blocker:
- [ ] 시스템 레시피북 이름 + count가 한 행에 표시되는지
- [ ] 커스텀 레시피북 이름 + ⋯ 버튼이 겹치지 않는지
- [ ] 탭 바 두 탭의 텍스트가 잘리지 않는지
- [ ] 장보기 카드 제목 + 뱃지가 한 행에 표시되는지
- [ ] 프로필 이미지 + 닉네임 + ⚙️ 간 간격이 충분한지

## 디자인 결정 사항

1. **프로필 섹션 스크롤 포함**: 프로필 섹션을 sticky로 고정하지 않고 탭 콘텐츠와 함께 스크롤하도록 설계했다. 프로필 영역이 80px 정도로 작지만, 레시피북 목록이 길어질 수 있으므로 콘텐츠 영역을 최대한 확보한다. 탭 바만 sticky로 프로필 스크롤 아웃 후 상단에 고정된다.

2. **탭 바 sticky**: 프로필 섹션이 스크롤 아웃되면 탭 바가 상단에 고정된다. 이는 현재 탭이 무엇인지 항상 인지할 수 있게 하고, 탭 전환을 위해 상단까지 스크롤할 필요가 없도록 한다.

3. **시스템 레시피북에 ⋯ 메뉴 미노출**: 시스템 레시피북(my_added/saved/liked)은 이름 변경, 삭제가 불가하므로 ⋯ 메뉴 자체를 노출하지 않는다. 이는 서버 403보다 앞서는 UI 수준 차단이다.

4. **레시피북 섹션 분리 (나의/커스텀)**: 화면정의서 SS 19의 "시스템 섹션 + 커스텀 섹션" 구조를 그대로 따른다. 섹션 헤더로 시각적 구분을 두고, 커스텀 섹션 하단에 생성 CTA를 배치한다.

5. **장보기 기록 cursor pagination**: 장보기 기록은 무한 스크롤로 cursor pagination을 사용한다. 스크롤 하단 도달 시 자동 로드하여 사용자 경험을 끊김 없이 유지한다.

6. **카드 그림자 --shadow-1**: 다른 리스트 화면(LEFTOVERS, COOK_READY_LIST)과 동일한 시각적 가벼움을 유지한다. 마이페이지는 정보 탐색 화면이므로 카드 간 시각적 분리만 충분하면 된다.

7. **CTA 위계**: [+ 새 레시피북]이 점선 카드 + --brand 텍스트로 가벼운 CTA다. 과도한 강조 없이 "추가 가능" 어포던스를 전달한다. 이는 화면정의서의 [+ 새 레시피북] 명세와 일치한다.

8. **장보기 완료 뱃지 색상**: 완료 = --olive (긍정/완료 의미), 진행 중 = --brand (활성 상태 의미). 이 색상 구분은 PLANNER_WEEK의 상태 뱃지 패턴과 일관된다.

9. **h8 prototype parity scope**: MYPAGE shell(프로필 + 탭 바 + 전체 레이아웃)은 prototype parity 후보이나, 탭 내부 콘텐츠(MYPAGE_TAB_RECIPEBOOK, MYPAGE_TAB_SHOPPINGLISTS)는 prototype-derived design이다. Stage 4 구현 시 shell은 Baemin prototype 참조, sub-tab 콘텐츠는 design-tokens + mobile-ux-rules 기반으로 설계한다.

## Anchor Screen 영향 분석

- **MYPAGE는 anchor screen이 아니다** (anchor-screens.md 확인).
- **Anchor extension 해당 여부**: 해당하지 않음. 기존 anchor screen(HOME, RECIPE_DETAIL, PLANNER_WEEK)에 영향 없음.
- **모바일 UX 리스크**: 중간. 신규 화면이지만 구조는 프로필 + 탭 + 리스트로 익숙한 패턴이다. CRUD 인터랙션(생성/이름 변경/삭제)이 새로 추가되므로 터치 타겟, 확인 다이얼로그, 에러 피드백에 주의가 필요하다.

## 접근성 노트

- **탭 바**: `role="tablist"`, 각 탭 `role="tab"`, `aria-selected`, 활성 탭 패널 `role="tabpanel"`
- **레시피북 카드**: `role="listitem"` (상위 `role="list"`), 레시피북 이름이 접근 가능한 레이블
- **⋯ 메뉴 버튼**: `aria-label="레시피북명 옵션 메뉴"`, `aria-haspopup="menu"`
- **드롭다운 메뉴**: `role="menu"`, 각 항목 `role="menuitem"`, Escape 키로 닫기
- **삭제 확인 다이얼로그**: `role="alertdialog"`, `aria-modal="true"`, 열릴 때 포커스 트랩
- **[+ 새 레시피북]**: `aria-label="새 레시피북 만들기"`
- **장보기 기록 카드**: `aria-label="제목 - 날짜범위 - 상태"`
- **장보기 무한 스크롤**: `aria-live="polite"` 영역에 로드 상태 안내
- **색상 대비**: --brand (#ED7470) on white는 WCAG AA large text 통과, 작은 텍스트에서는 --brand-deep 검토
- **터치 타겟**: 모든 인터랙티브 요소 최소 44x44px
