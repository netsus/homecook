**mobile-narrow 대응 규칙**

- 카드 내부 여백: `--space-4 (16px)` -> `--space-3 (12px)`
- [+ 재료 추가] -> [+ 추가] 버튼 텍스트 축약
- 카테고리 칩 rail: 칩 3개 + 우측 fade hint로 스크롤 가능성 표시
- 재료명 long text: 1줄 말줄임 (`text-overflow: ellipsis`)
- 폰트 크기/터치 타겟 최소 44px 유지
- 레이아웃 비율 mobile-baseline과 동일

```
│  홈  │ 플래너 │ *팬트리* │ 마이페이지│
└───────────────────────────────────────┘
```

### Unauthorized 상태 (비로그인으로 팬트리 탭 진입)

비로그인 사용자가 팬트리 탭을 누르면 기존 공통 로그인 게이트 모달(`LoginGateModal`)이 표시된다.
팬트리 화면 자체는 렌더링하지 않으며, 모달이 로그인 흐름을 담당한다.

```
┌───────────────────────────────────────┐
│  HOMECOOK                             │
├───────────────────────────────────────┤
│                                       │
│  ┌─────────────────────────────────┐  │  <- LoginGateModal (기존 공통 패턴)
│  │          ─────                  │  │     --panel bg, --radius-xl 상단
│  │                            [X]  │  │     icon-only close
│  │                                 │  │
│  │   로그인이 필요해요             │  │  <- title (text-lg)  --foreground
│  │                                 │  │
│  │   집에 있는 재료를 등록하면     │  │  <- body (text-sm)   --muted
│  │   장보기 때 자동으로 빼줘요     │  │
│  │                                 │  │
│  │   [로그인]                      │  │  <- primary CTA  --brand bg
│  │                                 │  │     border-radius: --radius-md
│  └─────────────────────────────────┘  │     shadow: --shadow-3
│                                       │
├───────────────────────────────────────┤
│  홈  │ 플래너 │  팬트리  │ 마이페이지│
└───────────────────────────────────────┘
```

**Unauthorized 규칙**

- 기존 `LoginGateModal` 공통 패턴 사용 (다른 보호 화면과 동일한 UX)
- return-to-action: `/pantry`
- [로그인] 탭 -> LOGIN 화면 -> 로그인 성공 -> `/pantry` 자동 복귀
- 모달 닫기([X]) 시 이전 화면(HOME 등)으로 복귀

## 컴포넌트 상세

### 1. 검색바

- **기본 상태**: placeholder "재료 검색", 돋보기 아이콘 좌측
- **입력 중**: 검색어 표시 + 우측 [X] clear 버튼
- **동작**: `GET /pantry?q={검색어}` 호출 (debounce 300ms)
- **토큰**: --surface bg, --muted placeholder, --foreground 입력 텍스트, --radius-xl border-radius

### 2. 카테고리 필터 칩 rail

- **구성**: [전체] + 서버 카테고리 (채소/육류/해산물/양념 등)
- **활성 칩**: --brand-soft bg, --brand text, --radius-full
- **비활성 칩**: --surface-fill bg, --text-2 text, --radius-full
- **스크롤**: localized horizontal scroll (칩이 뷰포트를 넘으면)
- **affordance**: 우측 fade gradient hint
- **동작**: `GET /pantry?category={카테고리}` 호출
- **touch target**: 칩 높이 36px + 상하 여백으로 44px 보장
- "전체"는 항상 첫 번째 칩, 탭 시 카테고리 필터 해제

### 3. 액션 버튼 행

- **[+ 재료 추가]**: primary 스타일 --brand bg + --surface text
  - 탭 -> 재료 추가 바텀시트 열림
- **[묶음 추가]**: secondary 스타일 --surface bg + --olive border+text
  - 탭 -> PANTRY_BUNDLE_PICKER 진입
- 두 버튼은 같은 행에 50:50 또는 flex 배치
- border-radius: --radius-md (12px)
- min height: 44px

### 4. 재료 카드

- **기본 상태**: 카테고리 아이콘(선택) + 재료 표준명 + 카테고리 라벨
- **선택 모드**: 좌측에 체크박스 표시
- **선택됨**: --brand-soft bg, --brand border (2px)
- **미선택**: --surface bg, --shadow-1
- **터치**: 기본 모드에서 카드 탭은 무동작 (팬트리는 보유 여부만, 상세 없음)
- **선택 모드에서 카드 탭**: 선택/해제 토글
- **토큰**: --radius-lg (16px), 내부 패딩 --space-4, 카드 간격 --space-2

### 5. 선택 모드 액션 바 (하단 고정)

- **[선택 삭제 (N)]**: --brand-deep bg, --surface text
  - 탭 -> 삭제 확인 모달
- **[이 재료로 레시피 보기]**: --olive bg, --surface text (선택 구현)
  - 탭 -> HOME으로 이동, 선택된 재료를 필터로 적용
- 하단 탭바 바로 위에 고정 (`z-index` 탭바보다 높게 설정)
- 탭바와의 경계: `--line` (1px separator)
- safe-area: 탭바가 safe-area를 포함하므로 액션 바는 탭바 top edge에 붙음
- 선택 모드 진입 시 slide-up 애니메이션

### 6. 재료 추가 바텀시트

- **헤더**: eyebrow 없음, title "재료 추가", helper copy "팬트리에 추가할 재료를 검색하세요", icon-only close
- **검색 입력**: 재료 마스터 검색 (GET /ingredients?q=...)
- **카테고리 rail**: localized horizontal scroll, --olive active
- **재료 체크리스트**: 다중 선택 가능
  - 이미 팬트리에 있는 재료는 "보유 중" 라벨 표시, 체크 불가
- **CTA**: [팬트리에 추가 (N)] --brand bg
- **시트 규격**: --panel bg, --radius-xl 상단, --shadow-3
- **API**: POST /pantry { ingredient_ids: [...] }

### 7. 하단 탭바

- 4탭 고정: 홈 / 플래너 / 팬트리 / 마이페이지
- 팬트리 활성 시: --brand 아이콘+라벨
- 비활성 탭: --muted 아이콘+라벨
- 높이: 56px + safe-area-inset-bottom
- bg: --panel
- border-top: --line

## 디자인 결정 사항

1. **재료 카드 형태**: 리스트형이 아닌 카드형 채택. 재료별 카테고리 아이콘과 라벨을 함께 표시하여 시각적 스캔성 향상. 재료 수가 많아지면 (50+) 리스트형으로 전환 검토 가능.

2. **선택 모드 진입 방식**: 헤더의 [선택] 텍스트 버튼으로 진입. long-press 선택도 가능하나 MVP에서는 명시적 진입만 지원.

3. **재료 추가 UI**: 바텀시트 패턴 채택. 팬트리 화면 컨텍스트를 유지하면서 재료를 검색/선택할 수 있음. 전체 화면 이동보다 집중 흐름에 적합 (mobile-ux-rules SS 6).

4. **[이 재료로 레시피 보기] 구현**: 화면정의서에 "(선택)"으로 표기된 기능. 선택 모드에서만 노출하여 primary flow를 방해하지 않음. HOME 이동 시 선택된 재료를 query parameter로 전달.

5. **카테고리 칩 rail vs 탭**: 칩 rail 채택. 카테고리가 동적이고 개수가 유동적이므로 칩 rail + localized scroll이 탭보다 유연함. INGREDIENT_FILTER_MODAL의 카테고리 rail 패턴과 일관성 유지.

6. **보유 재료 카운트 위치**: 화면 상단 제목 아래에 "N개 재료 보유 중" 표시. 섹션 헤더에도 카운트를 표시하되, 상단은 전체 수, 섹션 헤더는 필터된 수를 표시.

7. **prototype parity 참고**: h8 gate 기준 Baemin prototype을 시각 참고하되, 화면정의서 SS 17의 기능 범위를 벗어나는 요소는 추가하지 않음.

- [ ] 삭제 확인 모달의 문구("장보기 목록에서 자동 제외되지 않아요")가 사용자에게 의미 있는지

## 접근성 (a11y)

- 검색바: `role="searchbox"`, `aria-label="팬트리 재료 검색"`
- 카테고리 칩: `role="tablist"`, 각 칩 `role="tab"`, `aria-selected`
- 재료 카드 (선택 모드): `role="checkbox"`, `aria-checked`, `aria-label="재료명 선택"`
- 선택 모드 액션 바: `aria-live="polite"` (선택 수 변경 알림)
- 바텀시트: `role="dialog"`, `aria-modal="true"`, focus trap
- 하단 탭바: `role="navigation"`, 각 탭 `role="tab"`

## 성능 고려사항

- 팬트리 재료 수: 일반적으로 20~80개 수준, 가상 스크롤 불필요
- 검색 debounce: 300ms
- 낙관적 업데이트: 추가/삭제 시 즉시 UI 반영, 실패 시 롤백 + 에러 토스트
- 카테고리 칩 데이터: 서버에서 카테고리 목록을 별도 조회하지 않고, 팬트리 응답의 category 필드에서 추출하여 클라이언트에서 생성
