## 화면 개요

내부 운영 관리 대시보드. 관리자(`admin_members` 등록 운영자)만 접근 가능하며 일반 사용자 네비게이션에 노출되지 않는다. 사용자 통계 요약, 운영 이벤트 요약, 관리 화면 네비게이션을 제공한다.

- **권한**: 관리자 전용 (`admin_members` 등록 필수)
- **API**: `GET /api/v1/admin/users` (요약 count), `GET /api/v1/admin/operational-events` (최근 이벤트 count)
- **진입**: `/admin` 직접 접근 (일반 네비게이션에서 노출 안 함)
- **이탈**: Admin 내부 네비게이션 (사용자/이벤트/감사 로그) 또는 브라우저 뒤로가기
- **감사**: 진입 시 `admin_page_view` 감사 기록 — 렌더링 전에 기록, 실패 시 fail closed

## Admin 공통 네비게이션

모든 Admin 화면에 공통으로 적용되는 상단 네비게이션. 일반 앱 하단 탭바와 분리된 독립 네비게이션이다.

```
+---------------------------------------+  <- 375px (모바일 기준)
|  관리자                               |  <- 앱바 (56px)
+---------------------------------------+     "관리자": text-xl, 800, --foreground
|  대시보드  사용자  이벤트  감사로그   |     배경: --surface
+---------------------------------------+  <- Admin 탭바 (44px)
                                              탭: text-sm, 600
                                              활성: --brand, 하단 2px 보더 --brand
                                              비활성: --text-3
                                              터치 타겟: 44px 높이
                                              overflow-x: auto (모바일)
```

### 후속 네비게이션 placeholder

커뮤니티/신고/제재 탭은 현재 미구현. Admin 탭바에 disabled placeholder로 표시하되 **탭 불가, 네비게이션 없음**.

```
|  ... 감사로그  [커뮤니티]  [신고]     |  <- disabled placeholder
                                              --text-4, opacity 0.5
                                              cursor: default
                                              탭 시 동작 없음
                                              미구현 페이지로 이동하지 않음
```

## 320px 대응 (작은 모바일 sentinel)

- **앱바**: "관리자" 제목이 320px에서도 충분히 짧아 문제 없음
- **Admin 탭바**: 4개 활성 탭 + 2개 disabled placeholder가 320px에서 넘칠 수 있음 → overflow-x auto로 가로 스크롤 (탭바 내부만)
- **요약 카드**: 2열 그리드 유지. 각 셀의 숫자(text-2xl)와 라벨(text-xs)은 320px에서도 안정적
- **바로가기 행**: 텍스트가 짧아 wrap 불필요. chevron 우측 고정

320px에서 확인해야 할 blocker:
- [ ] Admin 탭바가 가로 스크롤 시 현재 활성 탭이 뷰포트 내에 보이는지
- [ ] 요약 카드 2열 그리드에서 숫자와 라벨이 겹치지 않는지
- [ ] 모든 터치 타겟이 44px 이상인지

## 디자인 결정 사항

1. **상단 탭바 네비게이션 채택 (사이드바 대신)**: Admin 화면이 4개(+ 2개 미구현 placeholder)로 적은 수이므로 사이드바보다 상단 탭바가 모바일 친화적이다. 사이드바는 모바일에서 햄버거 메뉴가 필요하고 화면 폭을 소비한다.

2. **요약 카드는 데이터 중심, 장식 최소화**: 내부 운영 도구이므로 그래프/차트 없이 핵심 수치만 표시한다. 숫자가 크게 보이되 카드 자체는 작고 밀도 높게 배치한다.

3. **바로가기 행 별도 제공**: Admin 탭바로도 이동 가능하지만, 대시보드에서 각 화면으로의 명시적 네비게이션을 제공하여 첫 진입 시 구조를 이해하기 쉽게 한다.

4. **일반 앱 하단 탭바 미표시**: Admin은 일반 사용자 네비게이션과 분리된 독립 화면이다. 하단 탭바를 보여주면 관리자가 일반 화면과 혼동할 수 있다.

5. **감사 기록 우선 (fail closed)**: admin_page_view 감사 기록이 실패하면 대시보드 데이터를 렌더링하지 않는다. 운영 가시성보다 감사 무결성이 우선이다.

## Anchor Screen 영향 분석

- **ADMIN_DASHBOARD는 anchor screen이 아니다** (anchor-screens.md 확인)
- **Anchor extension 해당 여부**: 해당하지 않음. Admin은 일반 사용자 네비게이션과 완전히 분리.
- **모바일 UX 리스크**: 낮음. 요약 카드 + 네비게이션 링크로 구성이 단순하다. 주요 리스크는 데스크톱 우선 설계가 모바일에서 불편해지는 것이나, 현재 구조는 모바일에서도 세로 스크롤만으로 동작한다.

## 접근성 노트

- **Admin 탭바**: `role="tablist"`, 각 탭 `role="tab"`, `aria-selected`, disabled placeholder는 `aria-disabled="true"`
- **요약 카드**: `role="group"`, 라벨과 숫자 `aria-label` 연결 ("총 사용자 150명")
- **바로가기 행**: `role="link"`, `aria-label` 포함
- **unauthorized/forbidden 상태**: 안내 메시지에 적절한 heading level
- **색상 대비**: --text-3 (#868E96) on --surface (#ffffff) = 3.7:1 (AA large text 통과)
- **터치 타겟**: 모든 인터랙티브 요소 최소 44x44px
