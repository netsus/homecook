## 화면 개요

관리자 전용 사용자 목록 읽기 전용 조회 화면. 마스킹된 이메일, 닉네임, 가입 프로바이더 등 승인된 요약 정보만 표시한다. 검색/필터/페이지네이션을 지원하되 검색어는 감사 로그에 기록하지 않는다.

- **권한**: 관리자 전용 (`admin_members` 등록 필수)
- **API**: `GET /api/v1/admin/users`
- **진입**: ADMIN_DASHBOARD 바로가기 또는 Admin 탭바 "사용자" 탭
- **이탈**: Admin 탭바로 다른 관리 화면 이동
- **감사**: API 호출 시 `action='list_users'`, 검색어 로그 미저장 (`target_type='user_search'`, `target_id=null`)

## PII 최소화 정책

이 화면에서 표시하는 사용자 정보:

| 필드 | 표시 | 비고 |
|------|------|------|
| `id` | UUID | 내부 식별용, 행 key |
| `email_masked` | `c***@example.com` | 서버에서 마스킹된 이메일만 수신 |
| `provider` | `google`, `kakao` | OAuth 프로바이더 |
| `nickname` | 닉네임 | 공개 정보 |
| `created_at` | 가입일 | YYYY-MM-DD 형식 |
| `recipe_count` | 레시피 수 | 고수준 카운트 |
| `status` | `active` / `inactive` | 계정 상태 |

**표시 금지**:
- 원문 이메일 주소
- OAuth 토큰
- YouTube URL / 소스 텍스트
- 비공개 장보기/팬트리 상세
- 관리자 검색어 (감사 로그에도 미기록)

| 검색어 초기화 | 검색 필드 clear 버튼 | 전체 목록으로 복귀, page 1 | GET /admin/users | admin |

### 스크롤 정책

| 영역 | 스크롤 방향 | 비고 |
|------|-----------|------|
| 앱바 + Admin 탭바 | 고정 (sticky) | 상단 고정 |
| 검색/필터 바 | 세로 스크롤 시 같이 이동 | 고정하지 않음 — 데이터 영역 확보 |
| 사용자 목록 | 세로 스크롤 | 테이블 또는 카드 리스트 |
| 페이지 전체 | 가로 스크롤 금지 | whole-page horizontal scroll 금지 |
| 테이블 (768px+) | 가로 스크롤 금지 | 열 비율 조정으로 맞춤 |

> **scroll containment**: 앱바 + Admin 탭바는 sticky로 고정되어 사용자 목록만 세로 스크롤된다. 검색/필터 바는 콘텐츠와 함께 이동하여 데이터 영역을 최대화한다. 테이블은 열 비율 조정으로 가로 스크롤 없이 맞춘다.

## 320px 대응 (작은 모바일 sentinel)

- **검색 필드**: placeholder 텍스트가 길면 말줄임. 검색 기능 자체는 동작
- **카드 뷰**: 카드 패딩 --space-3 → --space-2 축소. 닉네임은 말줄임
- **이메일**: 마스킹된 이메일이 긴 경우 말줄임 (`c***@very-long-domain...`)
- **메타 행**: 프로바이더 · 가입일 · 레시피 N이 한 행에 안 맞으면 2행 wrap 허용
- **페이지네이션**: 페이지 번호 5개 이상 시 [< 1 ... N >] 축약

320px에서 확인해야 할 blocker:
- [ ] 사용자 카드에서 닉네임 + 이메일 + 메타 정보가 읽히는지
- [ ] 검색 필드와 필터가 겹치지 않는지
- [ ] 페이지네이션 터치 타겟이 44px 유지되는지
- [ ] 카드 간격이 과도하게 좁아지지 않는지

---

## 디자인 결정 사항

1. **테이블/카드 반응형 전환**: 768px 이상에서는 데이터 밀도가 높은 테이블 뷰, 375px 이하에서는 읽기 편한 카드 뷰. 내부 운영 도구이므로 데스크톱에서는 밀도가 우선이고, 모바일에서는 스캔 가능성이 우선이다.

2. **검색어 감사 로그 미저장**: PII 보호를 위해 관리자의 검색어는 감사 로그에 기록하지 않는다. `target_type='user_search'`, `target_id=null`로 검색 행위 자체만 기록한다.

3. **프로바이더 pill 배지**: 프로바이더 구분을 위해 --olive 색상 pill을 사용한다. 프로바이더가 적은 수(google, kakao 등)이므로 색상 구분 없이 단일 색상이 적절하다.

4. **수정/삭제 UI 없음**: MVP 범위에서 사용자 관리 동작이 없으므로 행 클릭/상세 보기/동작 버튼을 일체 두지 않는다. 읽기 전용 목록만 제공한다.

## 접근성 노트

- **검색 필드**: `role="searchbox"`, `aria-label="사용자 검색"`
- **프로바이더 필터**: `role="combobox"` 또는 `<select>`, `aria-label="프로바이더 필터"`
- **테이블**: `role="table"`, 헤더 `role="columnheader"`, 행 `role="row"`
- **카드 뷰**: 각 카드 `role="article"`, heading level 적절히 배치
- **페이지네이션**: `nav` 랜드마크, `aria-label="페이지네이션"`, 현재 페이지 `aria-current="page"`
- **결과 카운트**: 검색/필터 변경 후 `aria-live="polite"`로 "총 N명" 갱신 안내
- **색상 대비**: --text-2 (#495057) on --surface (#ffffff) = 6.6:1 (AA 통과)
- **터치 타겟**: 검색 필드 44px, 필터 44px, 페이지네이션 각 44x44px
