## 공통 전제

- 기준 폭: 375px (모바일 기본), 390px (authority 캡처), 320px (narrow sentinel)
- 공통 요소: 브랜드 헤더(`HOMECOOK`), 하단 탭바(4탭), 레시피 카드(2열 그리드)

## 비교 요약

| 기준 | 안 A (유지) | 안 B (카탈로그) | 안 C (하이브리드) ★ |
|------|------------|----------------|---------------------|
| first viewport 레시피 카드 수 (390px) | ~2장 | ~4장 완전 + peek | ~0장 완전 + peek (carousel 아래) |
| 정렬 control proximity | ❌ 스크롤 아래 | ✅ 검색 행 | ✅ 필터 행 |
| 큐레이션 editorial feel | ✅ 강함 | ❌ 없음 | ✅ compact 유지 |
| 320px 레이아웃 안정성 | ✅ 검증됨 | ⚠️ 칩 잘림 리스크 | 🔶 검증 필요 (양호 예상) |
| carousel 가로 스크롤 도입 | ❌ | ❌ | ✅ (localized) |
| contract-evolution 범위 | ✅ 없음 | ⚠️ 큰 구조 변경 | 🔶 소규모 (§1 정렬+테마형태) |
| 익숙한 앱 패턴 fit | 🔶 보통 | ✅ 배달앱 패턴 | ✅ 레시피앱 패턴 |
| 구현 복잡도 | 낮음 | 낮음 | 중간 |

**권고: 안 C** — 탐색 효율(정렬 이동, 레시피 peek)과 브랜드 가치(큐레이션 유지)를 균형 있게 달성.
