## 비교 요약

| 기준 | 안 A (유지) | 안 B (카탈로그) | 안 C (하이브리드) ★ 확정 |
|------|------------|----------------|--------------------------|
| first viewport "모든 레시피 [정렬▾]" 가시 (390px) | ❌ 스크롤 아래 | ✅ 검색 행 옆 | ✅ carousel 축소로 진입 |
| 정렬 의미 일관성 | ✅ 섹션 헤더 | ❌ 검색 행 (연결 약함) | ✅ 섹션 헤더 유지 |
| 큐레이션 editorial feel | ✅ 강함 | ❌ 없음 | ✅ compact 유지 |
| 320px 레이아웃 안정성 | ✅ 검증됨 | ⚠️ 칩 잘림 리스크 | ✅ 단독 행이라 리스크 없음 |
| carousel 가로 스크롤 도입 | ❌ | ❌ | ✅ (localized) |
| contract-evolution 범위 | ✅ 없음 | ⚠️ 큰 구조 변경 | ✅ 최소 (§1 테마 형태만) |
| 익숙한 앱 패턴 fit | 🔶 보통 | ✅ 배달앱 패턴 | ✅ 레시피앱 패턴 |
| 구현 복잡도 | 낮음 | 낮음 | 낮음~중간 |

**확정: 안 C** — carousel strip 축소만으로 정렬 섹션 헤더가 first viewport에 진입하고, 정렬 의미 일관성도 유지됨.
