# baemin-prototype-parity-foundation Evidence

Foundation artifacts for scored parity slices. These docs define how captures are taken, scored, and stored so that every parity slice follows identical rules.

## Contents

| File | Purpose |
| --- | --- |
| `capture-recipe.md` | 3-way capture rules: viewports, fixture, route entry, state, naming |
| `fixture-route-matrix.md` | Per-surface fixture baseline, route entry, required states |
| `visual-verdict-schema.json` | Machine-readable artifact schema for visual-verdict results |
| `visual-verdict-template.md` | Human-readable verdict template for parity slice PRs |
| `token-material-mapping.md` | Prototype token to production token mapping scope |
| `prototype-exclusion-inventory.md` | Concrete list of excluded prototype-only assets/fonts/features |

## Authority

- Source contract: `docs/workpacks/h7-baemin-prototype-parity-direction/README.md`
- This foundation does not change h7 thresholds, score axes, or exclusions.
- Scored parity slices consume these artifacts; they do not redefine them.
