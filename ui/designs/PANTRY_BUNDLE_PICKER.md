# PANTRY_BUNDLE_PICKER -- 팬트리 묶음 추가 (세트 방식)

> 기준 문서: 화면정의서 v1.5.1 SS 18 / 요구사항 v1.6.4 SS 1-8 / 유저 Flow v1.3.1 SS 7
> API: v1.2.2 SS 11-4 (GET /pantry/bundles), SS 11-2 (POST /pantry)
> h8 classification: `prototype-derived design` -- Baemin vocabulary/material 사용, near-100% parity 아님
> 생성일: 2026-04-28

## 화면 메타

| 항목 | 값 |
|------|---|
| Screen/Modal ID | `PANTRY_BUNDLE_PICKER` |
| 표현 형태 | 바텀시트 (bottom sheet) |
| 권한 | 로그인 필수 (PANTRY 자체가 로그인 필수이므로 진입 시점에 이미 인증됨) |
| 진입 | PANTRY 화면의 [묶음 추가] 버튼 탭 |
| 목적 | "조미료 모음/야채 모음" 같은 카테고리에서 체크로 대량 추가 |
| Anchor screen | 아님 (HOME, RECIPE_DETAIL, PLANNER_WEEK만 anchor) |
| h8 gate | `prototype-derived design` -- parity 대상 아님 |

## 설계 결정: 바텀시트 선택 근거

mobile-ux-rules Rule 6에 따라 "짧은 선택, 필터, 부가 입력은 modal/bottom sheet를 우선 검토"한다.

- PANTRY_BUNDLE_PICKER는 묶음 선택 -> 재료 체크 -> 추가의 짧은 선택 흐름이다
- 사용자가 원래 보던 PANTRY 컨텍스트를 유지해야 한다 (추가 완료 후 즉시 복귀)
- 전체 페이지 이동보다 시트가 진입/이탈 비용이 낮다
- 참고 와이어프레임(session4)은 전체 화면을 사용했으나, 공식 화면정의서에서는 "Screen/Modal" 형태로 지정하여 시트도 허용

단, 묶음 수가 많고 재료 체크리스트가 길 수 있으므로 시트 높이를 `height: auto; max-height: 90vh`로 설정하여 컨텐츠에 맞추되 상한을 확보한다.

**mobile-narrow 대응 규칙**

- 시트 내부 수평 여백: `--space-4 (16px)` -> `--space-3 (12px)`
- 재료 행 내부 패딩: `--space-3 (12px)`로 축소
- 묶음 카드 헤더에서 재료 수 ("6개") 생략 가능, 아코디언 아이콘은 유지

- CTA 버튼: 터치 타겟 48px, 폰트 크기 text-base 유지
- 체크박스 touch target: 44x44px 유지 (행 전체가 터치 영역)

## 컴포넌트 상세

### 1. 바텀시트 컨테이너

- **bg**: --panel (rgba(255,252,248,0.92))
- **border-radius 상단**: --radius-xl (20px)
- **shadow**: --shadow-3
- **높이**: `height: auto` (content-adaptive), `max-height: 90vh`. 컨텐츠가 90vh를 초과할 때만 내부 스크롤 활성화. 묶음이 2-3개일 때 하단 빈 공간을 방지한다.
- **drag handle**: 40x4px, --line, border-radius 2px, 시트 상단 중앙
- **닫기**: drag-down, overlay 탭, [X] 버튼 모두 지원
- **닫기 시**: 선택 상태 폐기, PANTRY 복귀

### 2. 시트 헤더

- **title**: "묶음으로 추가" (text-lg, 700 weight) --foreground
- **helper copy**: "자주 쓰는 재료를 묶음으로 한번에 추가해요" (text-sm, 400 weight) --muted
- **닫기 버튼**: icon-only 원형 (X), 우상단 배치
  - touch target: 44x44px
  - 아이콘: 16px, --muted
  - eyebrow 없음 (v1.5.0 modal 디자인 방향 준수)

### 3. 묶음 카드 (아코디언)

- **접힘 상태**
  - 한 줄: 묶음명 (text-sm, 700 weight, --foreground) + 재료 수 (text-xs, --text-3) + 아코디언 아이콘 (--text-3)
  - bg: --surface
  - border-bottom: --line
  - 전체 행 높이: 48px (touch target 충족)
  - 아이콘: chevron-down (접힘) / chevron-up (펼침), 16px

- **펼침 상태**
  - 헤더는 동일 (아이콘 chevron-up으로 변경)
  - 하위 재료 체크리스트 노출
  - 한 번에 하나의 묶음만 펼침 (아코디언 단독 펼침)

### 4. 재료 행 (BundleIngredientRow)

- **행 컨테이너**: flex, items-center, gap: --space-3 (12px), padding: --space-3 horizontal + --space-3 vertical
  - border-bottom: --line
  - 전체 행이 터치 타겟 (min height: 44px)

- **미보유 재료** (is_in_pantry === false)
  - 기본 상태: **체크됨** (미보유를 추가하는 것이 주 목적)

## Scroll containment

| 영역 | 스크롤 축 | 범위 | 비고 |
|------|----------|------|------|
| 시트 내부 (묶음 리스트 영역) | 세로 | 헤더 아래 ~ CTA 위 | 메인 스크롤 (overscroll-behavior: contain) |
| 시트 배경 (PANTRY) | 없음 | - | overlay 활성 시 배경 스크롤 잠금 (body scroll lock) |
| 전체 페이지 가로 | 없음 | - | whole-page horizontal scroll 금지 |

**drag-down vs 내부 스크롤 우선순위**: 시트 내부 스크롤 영역의 `scrollTop === 0`일 때만 drag-down 제스처로 시트 닫기를 허용한다. `scrollTop > 0`일 때는 내부 스크롤이 우선하여 drag-down이 시트 닫기로 이어지지 않는다. 이는 표준 바텀시트 제스처 패턴이다.

## 디자인 결정 사항

1. **바텀시트 vs 전체 페이지**: 바텀시트 채택. mobile-ux-rules Rule 6에 따라 짧은 선택 작업은 시트가 적합. 참고 와이어프레임(session4)은 전체 페이지를 사용했으나, 공식 화면정의서가 "Screen/Modal"로 표기하여 시트를 허용하고 있으며, PANTRY 컨텍스트 유지가 사용자 흐름에 더 자연스럽다.

2. **아코디언 단독 펼침**: 한 번에 하나의 묶음만 펼침. 여러 묶음을 동시에 펼치면 시트 내부 스크롤이 길어져 CTA까지의 거리가 멀어지고, 사용자가 어느 묶음의 재료를 보고 있는지 혼란. 단독 펼침으로 집중도 유지.

3. **미보유 재료 기본 선택**: 묶음의 목적이 "집에 있는 것만 체크"이므로, 미보유 재료를 기본 선택 상태로 두어 사용자가 불필요한 것만 해제하는 방식 채택. 참고 와이어프레임과 동일한 방향.

4. **보유중 재료 강제 선택 허용**: 이미 보유 중인 재료도 사용자가 원하면 체크하여 추가할 수 있음. API가 중복 추가를 silent skip 처리하므로 안전. edge case이지만 사용자 자유도를 유지.

5. **CTA 라벨에 카운트 포함**: "[N개 팬트리에 추가]" 형태로 선택 수를 실시간 반영. 사용자가 얼마나 추가되는지 즉시 인지 가능.

6. **시트 높이 adaptive (max 90vh)**: `height: auto`로 컨텐츠에 맞추되 `max-height: 90vh`로 상한 제한. 묶음이 2-3개일 때 하단 빈 공간을 방지하고, 묶음이 많을 때는 90vh 상한 내에서 내부 스크롤. 하단 CTA는 시트 내부 하단 고정으로 항상 접근 가능.

7. **prototype-derived design 위치**: h8 gate에 따라 Baemin prototype의 vocabulary(카드형, 아코디언, 체크리스트)와 material(색상 토큰, 간격)을 사용하되, 레이아웃 구조는 near-100% parity가 아닌 독자 설계. 시트 패턴은 H5 modal-system-direction(v1.5.0)의 시트 규칙(eyebrow 없음, icon-only close)을 따른다.

## 접근성 (a11y)

- 시트: `role="dialog"`, `aria-modal="true"`, `aria-label="묶음으로 재료 추가"`, focus trap
- 묶음 카드 헤더: `role="button"`, `aria-expanded="true/false"`, `aria-controls="bundle-{id}-content"`
- 재료 행: `role="checkbox"`, `aria-checked="true/false"`, `aria-label="{재료명}"`, 행 전체가 터치 영역
- [보유중] pill: `aria-label="이미 보유 중"` (시각 보조 정보)
- CTA: `aria-disabled` (비활성 시), `aria-busy="true"` (추가 중)
- drag handle: `aria-hidden="true"` (시각 단서, 스크린 리더 불필요)
- overlay: `aria-hidden="true"` (배경 콘텐츠 차단)

## 성능 고려사항

- 묶음 수: 일반적으로 5-10개, 각 묶음 재료 4-15개. 가상 스크롤 불필요.

- 아코디언 애니메이션: CSS transition (height auto -> 0, 200ms ease)
