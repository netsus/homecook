# Slice 17 화면별 개선사항 메모

> 로컬 테스트 URL: `http://127.0.0.1:3100`
> 작성 방법: 화면을 직접 조작하면서 발견한 내용을 아래 형식에 맞춰 추가한다.

## 작성 규칙

- 한 문제는 한 항목으로 작성한다.
- 가능하면 재현 경로를 버튼/탭 이름 기준으로 적는다.
- 기대 동작과 실제 동작을 분리해서 적는다.
- 스크린샷이 있으면 파일 경로나 캡처명을 남긴다.
- 우선순위는 `P0/P1/P2/P3` 중 하나로 적는다.
  - `P0`: 진행 불가, 데이터 손상, 보안/권한 문제
  - `P1`: 핵심 기능 오작동
  - `P2`: 사용성 저하, 헷갈리는 UI
  - `P3`: 문구/간격/시각적 polish

## 공통 체크

- [ ] 모바일 좁은 화면에서 버튼/문구가 잘리지 않는다
- [ ] 하단 탭바와 콘텐츠가 겹치지 않는다
- [ ] 로딩/빈 상태/에러 상태가 자연스럽다
- [ ] 확인/취소 다이얼로그의 버튼이 명확히 보인다
- [ ] 실패 시 에러 메시지가 보이고 잘못된 상태로 고정되지 않는다
