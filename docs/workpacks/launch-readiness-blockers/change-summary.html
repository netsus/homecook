한 줄 결론

배포 직전에 막힐 수 있던 문제를 줄였습니다. 이제 기본 법적 문서가 있고, 검색엔진이 읽을 파일이 있으며, preview 도메인과 가짜 연락처가 사용자 화면에 그대로 나가지 않습니다.

법적 문서 사용자 공개 화면

개인정보처리방침과 이용약관 페이지를 추가했습니다

이전

/privacy, /terms 주소가 없어서 404가 날 수 있었습니다.

지금

두 페이지가 생겼고, 운영자/문의처/시행일 같은 값을 환경변수로 넣을 수 있습니다.

기대효과

사용자가 하단/푸터/검색 결과에서 개인정보처리방침과 이용약관을 열었을 때 빈 페이지가 아니라 실제 문서 화면을 봅니다. 모바일에서는 본문이 하단 탭에 가려지지 않고, 데스크톱에서는 운영 정보 카드와 본문을 나란히 읽을 수 있습니다.

SEO 검색엔진

robots.txt와 sitemap.xml을 추가했습니다

이전

검색엔진이 “어떤 페이지를 봐야 하는지” 알려주는 기본 파일이 없었습니다.

지금

홈, 로그인, 개인정보, 약관 주소가 sitemap에 들어가고 관리자/인증 내부 주소는 제외됩니다.

기대효과

배포 후 /robots.txt와 /sitemap.xml이 200으로 열립니다. 검색엔진은 공개 페이지 목록만 받아가고, 관리자 화면이나 인증 콜백 같은 내부 주소는 sitemap에 노출되지 않습니다.

도메인 준비 환경변수

preview 도메인을 코드에서 제거했습니다

이전

homecook-flame.vercel.app 같은 임시 주소가 metadata 기본값으로 들어갈 수 있었습니다.

지금

NEXT_PUBLIC_SITE_URL에 실제 도메인을 넣으면 canonical, robots, sitemap이 그 도메인을 씁니다.

기대효과

실제 도메인을 NEXT_PUBLIC_SITE_URL에 넣으면 브라우저 HTML의 canonical 주소, Open Graph 주소, robots sitemap 주소가 모두 같은 공식 도메인으로 맞춰집니다. 마케팅 링크를 공유했을 때 preview 도메인이 섞일 가능성이 줄어듭니다.

문의처 마이페이지

가짜 문의 이메일과 카카오 채널명을 제거했습니다

이전

support@homecook.local, @homecook 같은 임시 문구가 보였습니다.

지금

실제 문의 이메일이 설정되면 보여주고, 없으면 “정식 운영 정보 확정 후 공개”라고 표시합니다.

기대효과

사용자가 마이페이지 도움말에서 문의처를 볼 때 존재하지 않는 개발용 이메일이나 임시 카카오 채널명을 보지 않습니다. 운영 이메일이 들어가면 바로 mailto: 링크로 클릭 가능한 문의 주소가 표시됩니다.

테스트 재발 방지

출시 준비용 회귀 테스트를 추가했습니다

이전

법적 페이지, sitemap, robots, 가짜 연락처 제거가 계속 지켜지는지 자동 확인이 약했습니다.

초보 개발자를 위한 용어 설명

이번 작업에서 나온 단어를 쉬운 말로 풀면 아래와 같습니다.

Route

웹사이트 주소입니다. 예: /privacy는 개인정보처리방침 주소입니다.

SEO

검색엔진이 사이트를 잘 이해하고 보여주도록 준비하는 작업입니다.

robots.txt

검색엔진에게 “이 사이트를 이렇게 읽어도 된다”고 알려주는 파일입니다.

sitemap.xml

검색엔진에게 중요한 페이지 주소 목록을 알려주는 지도입니다.

canonical

검색엔진에게 “이 페이지의 공식 주소는 이것”이라고 알려주는 표시입니다.

환경변수

코드에 직접 박지 않고 서버 설정에서 넣는 값입니다. 도메인, 문의 이메일처럼 운영마다 달라지는 값에 씁니다.

audit

설치된 패키지에 알려진 보안 문제가 있는지 검사하는 명령입니다.

Lighthouse

웹페이지 성능을 자동으로 점검하는 도구입니다. 로딩 속도와 화면 안정성을 봅니다.

아직 사람이 해야 하는 일

코드로는 준비했지만, 실제 운영 정보는 사람이 확정해야 합니다.

  • 실제 도메인을 구매하고 DNS를 연결합니다.
  • 호스팅 환경변수에 NEXT_PUBLIC_SITE_URL을 실제 도메인으로 넣습니다.
  • 공개 문의 이메일을 확정하고 NEXT_PUBLIC_SERVICE_CONTACT_EMAIL에 넣습니다.
  • 운영자명, 개인정보 보호책임자, 시행일, 제3자 제공/위탁/국외 이전 문구를 법무 기준으로 검토합니다.
  • 실제 운영 데이터베이스와 실제 OAuth 설정으로 마지막 production smoke를 실행합니다.
