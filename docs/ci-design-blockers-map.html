Homecook MVP Design CI Map

디자인을 바꿀 때 CI가 어디서 막는지 보는 지도

집밥 MVP에서 앱/웹 디자인을 수정할 때 어떤 CI가 실행되고, 어떤 디자인 테스트가 blocker가 되는지 초보 개발자도 한눈에 볼 수 있게 정리했다. 기준은 현재 저장소의 GitHub Actions, Playwright, Vitest, Lighthouse 설정이다.

앱

모바일 화면

1024px 미만. Wave1 모바일 기준, 작은 화면, 바텀탭, 터치 크기가 중요하다.

웹

데스크탑 화면

1024px 이상. --web-* 토큰과 components/web primitive가 기준이다.

CI

자동 검사

PR에서 lint, typecheck, unit test, build, Playwright, Lighthouse, 정책 검사가 돈다.

Blocker

주요 차단점

스크린샷 차이, 접근성, 작은 화면 가림, 성능 예산, PR evidence 누락이 자주 막는다.

앱/웹 기준 CI 구성 디자인 테스트 차단 요소 명령어 근거 파일

1. 앱과 웹의 책임 분리

앱 디자인 트랙

  • 기준 화면: mobile-chrome, mobile-ios-small
  • 주요 토큰: --brand-primary, --radius-*, --control-height-*
  • 주요 기준: Wave1 mobile fixed prototype, 390px/320px reference
  • 흔한 실패: 터치 타깃 44px 미만, CTA 가림, 스냅샷 diff 초과

웹 디자인 트랙

  • 기준 화면: desktop-chrome
  • 주요 토큰: --web-brand, --web-r-*, --web-s-*
  • 주요 기준: components/web/REFERENCE_LOCK.md
  • 흔한 실패: desktop visual baseline, web primitive class/test drift

2. CI 구성

워크플로우 언제 실행되나 검사 내용 디자인 blocker 가능성
Policy
.github/workflows/policy.yml		 PR 열림, synchronize, ready for review, protected branch push 브랜치/커밋/workpack/source-of-truth/QA evidence/authority evidence 높음 PR evidence가 빠지면 코드가 맞아도 막힌다.
PR Governance
.github/workflows/pr-governance.yml		 PR 열림, 수정, 동기화 라벨 자동 부착, PR 본문 섹션 검증 중간 PR 템플릿 미작성 시 막힌다.
CI
.github/workflows/ci.yml		 app/**, components/**, lib/**, tests/**, package/config 변경 pnpm lint, pnpm typecheck, pnpm test, pnpm build 높음 타입, aria label, class, build 오류가 잡힌다.
QA
.github/workflows/playwright.yml		 app/globals.css, 주요 route, components/**, Playwright 설정 변경 smoke E2E, accessibility, visual regression, Lighthouse 매우 높음 디자인 변경을 직접 막는 핵심 CI다.
Security Smoke
.github/workflows/security-smoke.yml		 auth, API, login, 주요 화면, components 변경 pnpm test:e2e:security 중간 로그인 게이트와 return-to-action 주변에서 걸릴 수 있다.
QA Eval
.github/workflows/qa-eval.yml		 QA 시스템 문서/스크립트/fixture/workflow 변경 pnpm qa:eval:suite 낮음 일반 화면 디자인보다 QA 시스템 변경 때 중요하다.

3. 디자인 요소를 직접 보는 테스트

Visual

스크린샷 비교

tests/e2e/qa-visual.spec.ts가 실제 화면을 캡처하고 기준 PNG와 비교한다.

홈, 정렬 메뉴, 재료 모달, 레시피 상세, 로그인 게이트, 데스크탑 planner/shopping/account/cooking 등이 포함된다.

A11y

접근성 + 터치 기본선

tests/e2e/qa-a11y.spec.ts가 axe 위반, 버튼 높이, 글자 크기, 줄바꿈 정책을 본다.

핵심 버튼은 보통 최소 44px, 컨트롤 글자는 기본 14px 이상이다.

Lighthouse

성능 예산

lighthouserc.js가 /와 /recipe/mock-kimchi-jjigae를 측정한다.

Performance 0.7 이상, LCP 4500ms 이하, CLS 0.1 이하, TBT 900ms 이하가 기준이다.

App Unit

앱 primitive

tests/ui-primitives.test.tsx가 버튼, 칩, 카드, 모달의 class와 최소 터치 크기를 고정한다.

Web Unit

웹 primitive

tests/web-primitives.test.tsx가 WebButton, WebDialog, modal backdrop, porting ledger를 확인한다.

Typography

폰트 정책

tests/mobile-typography-policy.test.ts는 Jua, next/font/google, 임의 fontFamily 추가를 막는다.

4. 디자인 수정 때 막을 만한 요소

변경 내용 막는 검사 이유 먼저 볼 명령
앱 색상, radius, 버튼 높이 visual, a11y, primitive, Lighthouse 스크린샷 diff, 44px 터치 기준, CLS 문제가 생길 수 있다. pnpm test:e2e:a11y
pnpm test:e2e:visual
웹 --web-* 토큰 또는 components/web desktop visual, web primitive unit 데스크탑 reference lock과 snapshot이 흔들린다. pnpm test -- tests/web-primitives.test.tsx
버튼/링크 텍스트 또는 aria-label unit, smoke E2E, a11y 테스트가 getByRole, 버튼 이름, dialog 이름으로 요소를 찾는다. pnpm test
pnpm test:e2e:smoke
modal, sheet, sticky CTA visual, a11y, smoke, authority evidence 작은 화면에서 CTA가 가려지거나 dialog role/name이 깨질 수 있다. pnpm verify:frontend
폰트, 이미지, 큰 배경, 애니메이션 Lighthouse, visual, typography policy 리소스 예산, LCP, 스냅샷 안정성, 금지 폰트 정책에 걸릴 수 있다. pnpm test:lighthouse
PR 본문, QA evidence 누락 Policy, PR Governance 코드가 통과해도 PR 증거 조건을 만족해야 한다. pnpm validate:pr

5. 수정 전 추천 명령어

앱 디자인만 바꿀 때

pnpm test -- tests/ui-primitives.test.tsx
pnpm test:e2e:a11y
pnpm test:e2e:visual

웹 디자인만 바꿀 때

pnpm test -- tests/web-primitives.test.tsx
pnpm test:e2e:visual
pnpm test:lighthouse

6. 실패 메시지 빠른 해석

toHaveScreenshot

기준 이미지와 다르다. 의도한 변경이면 snapshot/evidence를 갱신한다.

axe violations

접근성 위반이다. 색 대비, role/name, label, focus 구조를 본다.

height 44

모바일에서 누르기 어려운 컨트롤이다. 버튼/칩 최소 높이를 본다.

getByRole

테스트가 버튼, 제목, dialog를 못 찾는다. 텍스트나 aria-label을 확인한다.

CLS

로딩 중 화면이 밀렸다. 이미지 크기와 skeleton 높이를 확인한다.

forbidOnly

test.only가 남아 있을 수 있다.

authority evidence

디자인 PR에 필요한 screenshot/report reference가 빠졌다.

frozen-lockfile

package.json과 lockfile이 맞지 않는다.
