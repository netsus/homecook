YouTube 레시피 Evidence Packet 추출 계획

유튜브 영상에서 재료와 조리 단계를 뽑을 때, 큰 프롬프트 하나에 맡기지 않고 증거 묶음을 먼저 만든 뒤 LLM과 비전을 단계적으로 쓰는 계획입니다.

!딱 한 줄 요약

영상 전체를 한 번에 LLM에게 던지지 말고, 레시피 후보별로 증거 봉투인 EvidencePacket을 만든 다음 그 봉투만 정리하게 하자는 계획입니다.

비유: 냉장고 속 재료를 전부 바닥에 쏟아놓고 요리법을 맞히게 하는 대신, “후토마끼 봉투”, “칼국수 봉투”처럼 요리별 봉투에 관련 재료와 메모를 나눠 담은 뒤 확인하는 방식입니다.

중요한 수정 gpt-5.5 검토 후, “텍스트만으로 모든 정답을 맞힌다”는 기대를 뺐습니다. source에 없는 180도 10분, 700ml, 상세 큰술 계량은 text cue 성공 기준이 아니라 visualFrameCues 보강 대상입니다.

1헷갈리는 구분

Evidence-first

  • 설명란, 댓글, 자막을 먼저 EvidenceRef로 정리합니다.
  • 레시피 후보마다 EvidencePacket을 만듭니다.
  • 텍스트에 있는 계량은 visual-estimate보다 우선합니다.

Vision-first

  • 영상을 통째로 보고 바로 정답 JSON을 만들게 합니다.
  • 비용과 실행 시간이 커질 수 있습니다.
  • 텍스트에 있는 값보다 시각 추정이 앞서면 정확도가 떨어질 수 있습니다.
source.json→ EvidenceRef→ EvidencePacket→ text-only prompt→ visualFrameCues→ result.json

2알아둘 용어

EvidencePacket 증거 봉투
한 레시피 후보에 관련된 제목, 자막, 계량, 단계 단서를 묶은 객체입니다.
EvidenceRef 증거 위치
어느 source의 몇 번째 줄, 몇 초 구간에서 나온 말인지 저장하는 참조입니다.
EvidenceCue 단서
재료, 계량, 단계, 온도, 시간처럼 LLM이 참고할 작은 힌트입니다.
visualFrameCues 시각 보강 단서
텍스트가 부족한 packet에만 selected frame이나 keyframe 결과를 넣는 보강 영역입니다.
visual-estimate 시각 추정
영상만 보고 추정한 계량입니다. 텍스트에 명시된 값보다 우선하면 안 됩니다.
packet boundary packet 경계
어떤 재료와 단계가 어느 레시피에 속하는지 나누는 선입니다.
deterministic cue extraction 규칙 기반 단서 추출
LLM 전에 정규식과 규칙으로 source에 실제 있는 단서만 뽑는 단계입니다.
augmentation 보강
부족한 정보를 비전 결과로 채우되, 원래 증거를 덮어쓰지 않는 추가 단계입니다.
baseline 기준 결과
새 로직이 최소한 이 정도는 유지해야 하는 기존 성능 기준입니다.
artifact 산출물 파일
디버깅과 검토를 위해 남기는 evidence-packets.json 같은 중간 결과입니다.

선택지장점문제판단

5데이터 모델

핵심은 정답 JSON을 바로 만들지 않고, 먼저 “증거 장부”를 만드는 것입니다.

type EvidenceSource = "description" | "comment" | "caption" | "transcript" | "visual";

type EvidenceCue = {
  kind: "title" | "ingredient" | "seasoning" | "amount" | "step" | "temperature" | "time" | "tip";
  basis: "source" | "visual";
  text: string;
  normalizedText?: string;
  refs: EvidenceRef[];
  confidence: number;
};

type EvidencePacket = {
  candidateId: string;
  titleHint: string;
  aliases: string[];
  timeRange: { startMs: number | null; endMs: number | null };
  titleEvidence: EvidenceRef[];
  ingredientCues: EvidenceCue[];
  stepCues: EvidenceCue[];
  amountCues: EvidenceCue[];
  visualFrameCues: EvidenceCue[];
  confidence: "high" | "medium" | "low";
  warnings: string[];
};
basis: "source"설명란, 댓글, 자막, transcript에서 나온 단서입니다. 명시 계량이면 우선권이 가장 높습니다.
basis: "visual"영상 프레임에서 나온 보강 단서입니다. 텍스트 단서를 덮어쓰면 안 됩니다.
금지lib/server/*가 scripts/*를 직접 import하지 않습니다. 필요한 규칙은 공통 순수 함수로 분리하거나 lab 모듈에 최소 이식합니다.

6실행 단계

  • P0

    Regression lock

    기존 fixture와 baseline을 기록합니다. 새 테스트는 production 같은 변경 전에 packet 구조를 먼저 고정합니다.

  • P1

    Timed transcript 보존

    buildSourceText()가 transcript를 한 덩어리로 합치며 잃던 startMs를 보존합니다.

  • P2

    Source evidence normalization

    source-evidence.mjs에서 description, author comments, transcript segments를 EvidenceRef[]로 통일합니다.

  • P3

    Candidate packet builder

    candidate-packets.mjs에서 description timeline과 title/tag를 이용해 4개 timeline row를 7개 child packet으로 나눕니다.

  • P4

    Deterministic cue extraction

    source에 실제 있는 cue만 뽑습니다. 없는 계량은 missingTextEvidence로 기록합니다.

  • P5

    Packet prompt

    buildExtractionPrompt()가 EvidencePacket[]을 받고, 첫 pass는 videoUrl: null인 text-only로 실행합니다.

  • P6

    Conditional vision augmentation

    confidence가 낮거나 cue가 부족한 packet에만 비전을 돌리고, 결과를 visualFrameCues artifact로 주입합니다.

  • P7

    Postprocess and provenance

    packet mode와 legacy mode를 분리하고, recoverLowTailVisualDefaults()가 source evidence를 덮지 못하게 막습니다.

  • P8

    Verification and rollout

    train target과 validation split을 검증한 뒤, production integration은 별도 계획에서 다룹니다.

7판단 규칙

timeline has explicit start timestimeline window를 primary candidate range로 사용합니다.
title has multiple dish namestimeline이 약하면 title candidates를 weak hint로 사용합니다.
candidate has no cooking cueconfidence: "low"로 표시하고 recipe output을 억지로 만들지 않습니다.
bundled candidate has child cues맥적&열무 들기름냉파스타처럼 parent range를 공유하는 child packet으로 나눕니다.
source에 없는 golden value180도 10분, 700ml 같은 값은 deterministic cue 성공으로 세지 않고 visualFrameCues 필요 항목으로 기록합니다.

10하지 않을 일과 안전장치

허용

  • lab 모듈 안에서 packet 생성과 artifact 저장을 실험합니다.
  • 기존 youtube-caption-normalizer.ts, youtube-multi-recipe-extractor.ts의 개념을 재사용합니다.
  • 텍스트 evidence가 부족하면 scoped vision artifact로 보강합니다.

금지

  • 이 계획에서 POST /recipes/youtube/extract를 바로 바꾸지 않습니다.
  • golden.json을 새 extractor에 맞춰 수정하지 않습니다.
  • 일반 요리 상식으로 없는 재료나 계량을 꾸며내지 않습니다.
  • 새 paid extraction provider를 추가하지 않습니다.

12GPT 5.5 검토 반영

검토 결론은 “Option C 방향은 맞지만, 텍스트 EvidencePacket만으로 fTlTpSJtrEs baseline 이상을 기대하면 무리”였습니다. 반영한 변경은 아래와 같습니다.

  • Phase 4 acceptance를 낮춰, source evidence에 실제 있는 cue만 성공으로 인정했습니다.
  • full fTlTpSJtrEs baseline parity를 conditional vision augmentation과 packet postprocess 이후 기준으로 옮겼습니다.
  • evidence-packets.json과 cue-extraction-report.json을 lab run artifact로 요구했습니다.
  • packet prompt 기본 pass는 videoUrl: null인 text-only로 실행하도록 했습니다.
  • vision은 별도 artifact를 만들고 visualFrameCues로만 주입하도록 했습니다.
  • legacy visual defaults가 source evidence를 덮지 않도록 packet-mode guardrail을 추가했습니다.
  • lib/server/*가 scripts/*를 직접 import하지 않도록 중복 통제 규칙을 넣었습니다.
