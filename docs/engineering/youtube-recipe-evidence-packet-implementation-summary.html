YouTube 레시피 Evidence Packet 구현 완료 요약

계획서에서 정한 “증거 봉투 먼저 만들기” 구조를 lab 추출 경로에 구현하고, 테스트와 빌드까지 확인한 내용을 정리했습니다.

!딱 한 줄 요약

유튜브 영상 전체를 바로 LLM에게 맡기기 전에, 레시피별 증거 봉투인 EvidencePacket을 만들고 그 중간 결과를 파일로 남기게 했습니다.

비유: 장보기 영수증과 조리 메모를 전부 섞어 두는 대신, “후토마끼 봉투”, “칼국수 봉투”처럼 요리별 봉투를 먼저 만든 뒤 요약하는 방식입니다.

1무엇이 바뀌었나

source.json
  -> source-evidence.mjs
  -> candidate-packets.mjs
  -> evidence-packets.json / cue-extraction-report.json
  -> packet-aware prompt
  -> extract.mjs normalization and postprocess
Production API 영향 이번 변경은 recipe-extraction-lab와 recipe-loop 도구 경로입니다. POST /recipes/youtube/extract 동작은 바꾸지 않았습니다.

2알아둘 용어

EvidenceRef 증거 위치
어느 source의 몇 번째 줄, 몇 초에서 나온 말인지 저장합니다.
EvidenceCue 작은 단서
재료, 계량, 단계처럼 LLM이 참고할 단서입니다.
EvidencePacket 증거 봉투
한 요리 후보에 관련된 증거와 단서를 모은 객체입니다.
cue report 단서 보고서
packet마다 어떤 단서가 있고 무엇이 부족한지 보여주는 JSON입니다.
source-visible cue 원문에 있는 단서
텍스트나 자막에 실제로 있는 단서입니다. 없는 값은 만들지 않습니다.
visualFrameCues 시각 보강 단서
나중에 selected frame 결과를 넣기 위한 보강 자리입니다.
text-only pass 텍스트만 보는 1차 실행
Gemini 기본 packet prompt에서 videoUrl: null로 실행하는 방식입니다.
artifact 중간 산출물
디버깅을 위해 남기는 evidence-packets.json 같은 파일입니다.

6GPT 5.5 검토 반영

검토 지적반영 내용
source에 없는 180도 10분, 700ml까지 Phase 4에서 기대했다.source-visible cue만 deterministic 성공으로 인정합니다.
llm.generate는 packet별 video scoping이 어렵다.Gemini 기본 packet prompt pass는 videoUrl: null text-only로 실행합니다.
기존 vision client와 중복 위험이 있다.lib/server/*에서 scripts/*를 직접 import하지 않았습니다.
legacy visual recovery가 text precedence를 덮을 수 있다.source text에 명시 amount가 있으면 visual default가 덮지 못하게 guard를 추가했습니다.

7남은 리스크
