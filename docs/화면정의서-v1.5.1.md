## 0) 전역 규칙

### 0-1. 하단 탭(IA)

- **홈(HOME) / 플래너(PLANNER_WEEK) / 팬트리(PANTRY) / 마이페이지(MYPAGE)**

### 0-2. 공통 상태(리스트/데이터 화면)

- **Loading**: 스켈레톤/로딩 인디케이터
- **Empty**: 빈 상태 안내 + 다음 행동 CTA
- **Error**: 오류 안내 + [다시 시도]

### 0-3. 로그인 게이트(전역 패턴)

- 비로그인 사용자가 로그인 필요 기능(좋아요/저장/플래너/팬트리/장보기 완료/요리 완료 등)을 시도하면:
    1. 즉시 로그인 이동 X
    2. 안내 모달 + [로그인] [취소]
    3. 로그인 완료 후 **원래 하려던 작업으로 자동 복귀(return-to-action)**

### 0-4. 핵심 데이터 용어(화면에서 쓰는 개념)

- **Recipe(레시피)**: 재료+스텝+기본 인분
- **Meal(식사)**: 플래너에 등록된 "식사(레시피 + 계획 인분)"
- **MealStatus(식사 상태)**
    - `식사 등록 완료` → `장보기 완료` → `요리 완료`
- **ShoppingList(장보기 리스트)**: 취합 결과 체크리스트(저장됨)
- **Pantry(팬트리)**: 재료 보유 여부만(수량 저장 X)

### 0-5. 조리방법 시각 규칙(요리모드 스텝 카드)

- 볶기/볶아주기: **주황 계열**
- 끓이기/국물: **빨간 계열**
- 굽기/오븐/구이: **갈색 계열**
- 찌기/스팀: **파란 계열**
- 튀기기: **노란 계열**
- 데치기: **연두 계열**
- 무치기/버무리기: **초록 계열**
- 섞기/준비/기타: **회색 계열**

### 0-6. 장보기 합산 표시 규칙(변환 불가 단위)

- 변환 가능(kg↔g, L↔ml): 변환 후 합산
- 변환 불가(개 vs g, 망 vs g 등): **라인 분리 없이 "복합 표기만"**
    - 예: `양파 2개 + 200g + 1망`

### 0-7. 장보기 상세 공통 정책 `v1.2 추가`

- 장보기 목록 생성 후 유저는 `SHOPPING_DETAIL`로 즉시 이동한다.
- `SHOPPING_DETAIL`은 **구매 섹션 / 팬트리 제외 섹션** 두 영역으로 구성된다.
- 미완료 리스트에서는:
    - 구매 체크
    - 팬트리 제외/되살리기
    - 카드 순서 변경(드래그&드롭)

    이 가능하다.

- 팬트리 제외 섹션으로 이동한 아이템은 자동으로 **구매 체크 해제**
- 장보기 완료 후 리스트는 **read-only**
    - 재열람은 가능하지만 체크/정렬/제외 수정은 불가
- 공유 텍스트는 **구매 섹션 항목만 포함**하고 제외 섹션은 포함하지 않는다.

## 1) HOME: 홈(레시피 탐색)

- **Screen ID**: `HOME`
- **권한**: 비로그인 가능
- **목적**: 레시피 탐색 → 레시피 상세 진입

### UI 구성

1. 공통 브랜드 헤더 (`HOMECOOK` 로고, `/` 링크)
2. discovery panel
    - 상단 검색바(placeholder: "레시피 제목 검색")
    - [재료로 검색] 버튼 → `INGREDIENT_FILTER_MODAL` (검색바 아래 단독 행)
3. 테마 carousel strip
    - compact horizontal carousel strip (높이 ~130px, 카드 1.5개 peek)
    - 가로 스크롤 affordance: 마지막 카드 0.5장 노출 + 우측 gradient hint
    - 헤더: "이번 주 인기" + "더보기 >" (tertiary CTA)
    - v1.3.1까지의 2열 카드 그리드 방식은 이 버전에서 폐기
4. `모든 레시피` 섹션
    - 카드 수
    - 정렬 컨트롤(기본: 조회수순)
5. 레시피 그리드(카드)

- 정렬 컨트롤은 검색바 아래가 아니라 `모든 레시피` 섹션 헤더에 둔다.

### 레시피 카드 정보

- 썸네일, source badge, 태그, 제목, 기본 인분 pill, 조회수, 좋아요 수, 저장 수

### 상호작용

- 카드 탭 → `RECIPE_DETAIL`
- 정렬 변경 → `모든 레시피` 리스트만 즉시 재정렬 (테마 섹션은 유지)
- 제목 검색 → 결과 리스트 갱신
- 재료 필터 적용 → "선택 재료가 모두 포함된 레시피"만 노출

### 정렬 선택 UI `v1.5.0 추가`

- **mobile**: bottom sheet 패턴
  - eyebrow 없음
  - title: `정렬 기준`
  - helper copy: `모든 레시피 순서를 바꿔요`
  - 닫기 버튼: icon-only 원형 버튼
  - 옵션 탭 시 즉시 적용 후 닫힘
- **desktop**: lightweight dropdown 패턴
  - mobile과 동일한 option language 사용
  - dark fill 대신 olive tint selected state 사용
  - `현재 {label}` badge는 사용하지 않는다
- 정렬 선택 UI는 `모든 레시피` 섹션을 위한 컨트롤이며, discovery 영역과 분리한다

### 상태

- Empty: "조건에 맞는 레시피가 없어요" + [필터 초기화]
- Error: "레시피를 불러오지 못했어요" + [다시 시도]

  - helper copy: `원하는 재료를 골라 레시피를 좁혀요`

## 3) RECIPE_DETAIL: 레시피 상세

- **Screen ID**: `RECIPE_DETAIL`
- **권한**
    - 조회/공유/요리모드 진입: 비로그인 가능
    - 좋아요/저장/플래너 추가: 로그인 필요
- **목적**: 레시피 정보 확인 + 액션 + 요리하기 진입

### UI 구성

1. 공통 브랜드 헤더 (`HOMECOOK` 로고, `/` 링크)
2. 미디어(영상/이미지)
3. breadcrumb + 제목 + 태그
4. overview meta: 기본 인분 / 재료 수 / 조리 단계 수
5. 보조 액션 row: `플래너 등록수 / 공유 / 좋아요 / 저장`
6. helper copy
7. primary CTA row: `[플래너에 추가] [요리하기]`
8. **인분 조절(가능)**: 스테퍼
9. 재료 리스트(상세 인분 반영)
10. 스텝 리스트

### 인분 조절 규칙

- 레시피 상세에서 인분을 변경하면 **재료량 표기가 즉시 변경**
- **요리모드에서는 인분 조절 불가** → 요리모드 진입 전 확정된 값으로 진행

### 버튼 동작

- 공유: Web Share → 실패 시 링크 복사
- 플래너 등록수: 읽기 전용 지표로 보조 액션 row에 노출
- 플래너 추가(로그인): PlannerAddPopup 바텀시트 열기 (아래 §PlannerAddPopup 참조)
- 저장(로그인):
    - 저장 대상은 **저장한 레시피북(saved)** 또는 **커스텀 레시피북(custom)** 만 허용
    - 레시피북 선택/생성 후 저장
- 좋아요(로그인): 토글
- 요리하기: `COOK_MODE` 진입(독립 요리 시나리오)
    - 상세에서 바로 요리한 경우: 남은요리 저장/팬트리 업데이트는 가능(로그인 시)
    - 플래너 식사 상태 변경은 수행하지 않음

### PlannerAddPopup `v1.5.0 갱신`

`[플래너에 추가]` 탭 시 열리는 바텀시트.

**입력 필드**

| 필드 | 설명 |
|------|------|
| 헤더 | eyebrow 없음 / title `플래너에 추가` / helper copy `날짜와 끼니를 선택해 주세요` / icon-only close |
| 날짜 선택 | compact date chip rail. chip 내부 표기 = `요일 + 4/17` |
| 끼니 선택 | 아침 / 점심 / 간식 / 저녁 segmented row |
| 인분 | compact stepper, 숫자와 단위 hierarchy 분리 (`2` + `인분`) |

**성공 동작** `v1.3.1 확정`

- 바텀시트 닫힘
- 토스트 표시: `N월 D일 끼니에 추가됐어요` (예: `4월 17일 아침에 추가됐어요`)
- **PLANNER_WEEK 이동 없음** — RECIPE_DETAIL 탐색 흐름 유지

**오류 동작**

- 토스트: `추가에 실패했어요. 다시 시도해 주세요`

**표시 규칙** `v1.5.0 추가`

- 별도 날짜 확인 라벨은 사용하지 않는다
- 날짜 / 끼니 / 인분 섹션은 같은 리듬의 선택 섹션처럼 읽혀야 한다
- accent는 `olive base + thin orange highlight`
- orange는 `today` hint, 작은 포인트 강조에만 제한한다

**비로그인 처리**

- `[플래너에 추가]` 탭 → 로그인 게이트 모달
- 로그인 성공 후 PlannerAddPopup 자동 복귀 (return-to-action)

**API 계약**

- `POST /meals` — 변경 없음. 기존 계약 그대로.
- body: `{ recipe_id, plan_date, column_id, planned_servings }`

- 토스트 "저장됐어요"

**API 계약**

    1. 플래너 경유일 경우 해당 세션 대상 식사들의 상태 → `요리 완료`
    2. 남은요리 자동 추가
    3. 소진 재료 체크리스트 팝업(기본 체크 해제)
    4. 체크된 재료만 팬트리에서 제거
    5. 완료 후 이전 화면으로 복귀
- 독립 요리의 경우:
    - 남은요리/팬트리 반영은 가능
    - 플래너 식사 상태는 변경하지 않음

## 20) RECIPEBOOK_DETAIL: 레시피북 상세

- **Screen ID**: `RECIPEBOOK_DETAIL`
- **권한**: 로그인 필요
- **목적**: 레시피북 안의 레시피 리스트 관리

### UI 구성

- 레시피 리스트(카드)
- 액션:
    - 레시피 탭 → `RECIPE_DETAIL`
    - 레시피 제거

### 제거 정책

- **좋아요한 레시피북(liked)** 에서 제거
    - 좋아요 해제 의미
- **저장한 레시피북(saved)** / **커스텀 레시피북(custom)**
    - 레시피북에서 제거 가능
- **내가 추가한 레시피북(my_added)**
    - 레시피북에서 제거 불가
    - 필요 시 별도 레시피 삭제 정책으로 처리(Phase 1 범위 밖 가능)
