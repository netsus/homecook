## 0) 전역 규칙

### 0-1. 하단 탭(IA)

- **홈(HOME) / 플래너(PLANNER_WEEK) / 팬트리(PANTRY) / 마이페이지(MYPAGE)**

### 0-2. 공통 상태(리스트/데이터 화면)

- **Loading**: 스켈레톤/로딩 인디케이터
- **Empty**: 빈 상태 안내 + 다음 행동 CTA
- **Error**: 오류 안내 + [다시 시도]

### 0-3. 로그인 게이트(전역 패턴)

- 비로그인 사용자가 로그인 필요 기능(좋아요/저장/플래너/팬트리/장보기 완료/요리 완료 등)을 시도하면:
    1. 즉시 로그인 이동 X
    2. 안내 모달 + [로그인] [취소]
    3. 로그인 완료 후 **원래 하려던 작업으로 자동 복귀(return-to-action)**

### 0-4. 핵심 데이터 용어(화면에서 쓰는 개념)

- **Recipe(레시피)**: 재료+스텝+기본 인분
- **Meal(식사)**: 플래너에 등록된 “식사(레시피 + 계획 인분)”
- **MealStatus(식사 상태)**
    - `식사 등록 완료` → `장보기 완료` → `요리 완료`
- **ShoppingList(장보기 리스트)**: 취합 결과 체크리스트(저장됨)
- **Pantry(팬트리)**: 재료 보유 여부만(수량 저장 X)

### 0-5. 조리방법 시각 규칙(요리모드 스텝 카드)

- 볶기/볶아주기: **주황 계열**
- 끓이기/국물: **빨간 계열**
- 굽기/오븐/구이: **갈색 계열**
- 찌기/스팀: **파란 계열**
- 튀기기: **노란 계열**
- 데치기: **연두 계열**
- 무치기/버무리기: **초록 계열**
- 섞기/준비/기타: **회색 계열**

### 0-6. 장보기 합산 표시 규칙(변환 불가 단위)

- 변환 가능(kg↔g, L↔ml): 변환 후 합산
- 변환 불가(개 vs g, 망 vs g 등): **라인 분리 없이 “복합 표기만”**
    - 예: `양파 2개 + 200g + 1망`

### 0-7. 장보기 상세 공통 정책 `v1.2 추가`

- 장보기 목록 생성 후 유저는 `SHOPPING_DETAIL`로 즉시 이동한다.
- `SHOPPING_DETAIL`은 **구매 섹션 / 팬트리 제외 섹션** 두 영역으로 구성된다.
- 미완료 리스트에서는:
    - 구매 체크
    - 팬트리 제외/되살리기
    - 카드 순서 변경(드래그&드롭)
        
        이 가능하다.
        
- 팬트리 제외 섹션으로 이동한 아이템은 자동으로 **구매 체크 해제**
- 장보기 완료 후 리스트는 **read-only**
    - 재열람은 가능하지만 체크/정렬/제외 수정은 불가
- 공유 텍스트는 **구매 섹션 항목만 포함**하고 제외 섹션은 포함하지 않는다.

## 1) HOME: 홈(레시피 탐색)

- **Screen ID**: `HOME`
- **권한**: 비로그인 가능
- **목적**: 레시피 탐색 → 레시피 상세 진입

### UI 구성

1. 상단 검색바(placeholder: “레시피 제목 검색”)
2. 검색바 아래 퀵 필터
    - [재료로 검색] 버튼 → `INGREDIENT_FILTER_MODAL`
    - 정렬 드롭다운(기본: 조회수순)
3. 테마 섹션(예: 인기/간단 한끼/홈파티 등)
4. 레시피 그리드(카드)

### 레시피 카드 정보

- 썸네일, 제목, 조회수, 좋아요 수, 저장 수, 태그

### 상호작용

- 카드 탭 → `RECIPE_DETAIL`
- 정렬 변경 → 즉시 재정렬
- 제목 검색 → 결과 리스트 갱신
- 재료 필터 적용 → “선택 재료가 모두 포함된 레시피”만 노출

### 상태

- Empty: “조건에 맞는 레시피가 없어요” + [필터 초기화]
- Error: “레시피를 불러오지 못했어요” + [다시 시도]

## 3) RECIPE_DETAIL: 레시피 상세

- **Screen ID**: `RECIPE_DETAIL`
- **권한**
    - 조회/공유/요리모드 진입: 비로그인 가능
    - 좋아요/저장/플래너 추가: 로그인 필요
- **목적**: 레시피 정보 확인 + 액션 + 요리하기 진입

### UI 구성

1. 미디어(영상/이미지)
2. 제목 + 태그(AI 자동 생성)
3. 지표: 조회수/좋아요/저장/플래너등록/요리완료 수
4. **인분 조절(가능)**: 스테퍼
5. 버튼 영역:
    - [공유] [플래너에 추가] [좋아요] [저장] [요리하기]
6. 재료 리스트(상세 인분 반영)
7. 스텝 리스트

### 인분 조절 규칙

- 레시피 상세에서 인분을 변경하면 **재료량 표기가 즉시 변경**
- **요리모드에서는 인분 조절 불가** → 요리모드 진입 전 확정된 값으로 진행

### 버튼 동작

- 공유: Web Share → 실패 시 링크 복사
- 플래너 추가(로그인): 날짜/끼니 선택 + 계획 인분 입력 → Meal 생성(status=`식사 등록 완료`)
- 저장(로그인):
    - 저장 대상은 **저장한 레시피북(saved)** 또는 **커스텀 레시피북(custom)** 만 허용
    - 레시피북 선택/생성 후 저장
- 좋아요(로그인): 토글
- 요리하기: `COOK_MODE` 진입(독립 요리 시나리오)
    - 상세에서 바로 요리한 경우: 남은요리 저장/팬트리 업데이트는 가능(로그인 시)
    - 플래너 식사 상태 변경은 수행하지 않음

### 상태

- 비로그인 시 좋아요/저장/플래너 추가 탭 → 로그인 게이트 모달
- `user_status`가 없는 경우에도 버튼은 노출하되, 클릭 시 로그인 안내

## 5) PLANNER_WEEK: 식단 플래너(위클리)

- **Screen ID**: `PLANNER_WEEK`
- **권한**: 로그인 필요
- **목적**: 식사(Meal) 계획 + 상태 기반 진행(장보기/요리)

### UI 구성

1. 상단 고정 버튼: **[장보기] [요리하기] [남은요리]**
2. 위클리 그리드(세로=날짜, 가로=끼니 컬럼)
3. 끼니 컬럼 관리
    - [+] 컬럼 추가
    - 최대 5개 제한
4. 날짜 이동
    - 위/아래 무한 스크롤

### 셀(식사) 표시

- 레시피명 + 계획 인분 + 상태 뱃지
    - 식사 등록 완료 / 장보기 완료 / 요리 완료
- 남은요리 기반 식사는 별도 시각 강조 가능
- 셀 탭 → `MEAL_SCREEN`

### 상호작용

- 빈 셀 탭 시 `MENU_ADD`로 바로 이동 가능(디자인 단계 선택 사항)
- 상단 [장보기] → `SHOPPING_FLOW`
- 상단 [요리하기] → `COOK_READY_LIST`
- 상단 [남은요리] → `LEFTOVERS`

## 6) MEAL_SCREEN: 끼니 화면(해당 날짜/끼니의 식사 목록)

- **Screen ID**: `MEAL_SCREEN`
- **권한**: 로그인 필요
- **목적**: 해당 슬롯의 식사 목록 관리

### UI 구성

- 헤더: 날짜 + 끼니명
- 식사 리스트(카드)
    - 레시피명, 계획 인분, 상태 뱃지
    - [인분 조절] (가능)
    - [삭제] (확인 모달 필수)
- 하단: [식사 추가] → `MENU_ADD`

### 인분/상태 변경 주의

- MealStatus가 `장보기 완료` 또는 `요리 완료`인 식사의 인분 변경 시:
    - 확인 모달: “상태가 진행된 식사입니다. 인분 변경 시 다시 장보기/요리 흐름이 필요할 수 있어요.”
    - MVP 처리: 변경 허용 + 상태는 그대로 유지

### 정책

- 이 화면에서는 개별 식사 [요리하기] 버튼을 제공하지 않는다.
- 식사 카드는 상세 조회용 진입 역할이다.

## 10) YT_IMPORT: 유튜브 레시피 가져오기/등록 `v1.2 보완`

- **Screen ID**: `YT_IMPORT`
- **권한**: 로그인 필요
- **목적**: URL → 자동 추출/표준화/추론 → 검수/수정 → 레시피 등록 → 플래너 추가

### 처리 파이프라인

1. 설명란/고정댓글 파싱 → 표준화
2. OCR(재료표 화면)
3. ASR 전사
4. 추정 레이어 고도화
    - 관용량 테이블 → 유사 레시피 Retrieval → 수량 예측 모델

### UI 흐름

- Step 1) URL 입력 + [가져오기]
- Step 1.5) 레시피 영상 검증
    - 레시피 영상이면 다음 단계 진행
    - 레시피 영상이 아닌 것 같으면:
        - 안내 문구 노출
        - 버튼: [다시 입력] [그래도 진행]
- Step 2) 자동 추론 진행 상태(단계별 Progress 표시)
    - extraction methods(`description`, `ocr`, `asr`, `estimation`) 표시 가능
- Step 3) 결과 검수/수정
    - 기본 인분(필수)
    - 레시피명
    - 재료 리스트(편집 가능)
    - 스텝(편집 가능)
    - 조리방법 자동 분류 결과 표시
    - 새로 생성된 조리방법은 “신규” 또는 유사한 라벨로 구분 가능
- Step 4) [레시피 등록] → 완료 후 “이 끼니에 추가”(계획 인분 입력) → Meal 생성

### 정책

- 수동 입력은 **검수 단계에서만** 허용
- 미분류 조리방법은 추출 단계에서 먼저 생성되어 선택지로 활용 가능

## 11) SHOPPING_FLOW: 장보기 생성(PLANNER_WEEK의 [장보기]) `v1.2 변경`

- **Screen ID**: `SHOPPING_FLOW`
- **권한**: 로그인 필요
- **목적**: `식사 등록 완료` 상태 식사 자동 취합 → 장보기 리스트 생성

### Step A) 대상 자동 취합

- 범위: **오늘 ~ 마지막 등록일**
- 대상:
    - 상태가 `식사 등록 완료`인 식사만

    - 체크/정렬/팬트리 제외 이동 가능
2. **읽기 전용 모드(read-only)**
    - 완료된 장보기 리스트
    - 마이페이지 장보기 기록에서 재열람 시 기본적으로 이 모드

### UI 구성

### A. 상단 영역

- 리스트 제목
- 대상 기간
- 액션 버튼:
    - [공유(텍스트)]
    - [쿠팡/컬리(검색 링크)] (선택 구현)
    - 편집 모드일 때 **[장보기 완료]**

### B. 구매 섹션(상단 본문)

- 실제 장볼 아이템 카드 리스트
- 각 카드:
    - 왼쪽 체크박스(구매 완료)
    - 재료명
    - 수량 표기
    - 카테고리/보조 정보
- 카드 드래그&드롭으로 순서 변경 가능
- 변환 불가 단위는 **복합 표기만**

### C. 팬트리 제외 섹션(하단 본문)

- 팬트리에 있어 자동 제외된 항목 리스트
- 유저가 되살리고 싶으면 다시 구매 섹션으로 이동 가능

### 아이템 이동 규칙

- 구매 섹션 → 팬트리 제외 섹션 이동:
    - `is_pantry_excluded=true`
    - **자동으로 구매 체크 해제**
- 팬트리 제외 섹션 → 구매 섹션 이동:
    - `is_pantry_excluded=false`
    - 다시 구매 대상에 포함

### [장보기 완료] 동작

1. 유저가 [장보기 완료] 클릭
2. “팬트리에 추가할 아이템 선택” 팝업 오픈
    - 표시 대상: 구매 체크된 아이템
    - 기본값: 전체 선택
    - 체크 해제로 팬트리 추가 제외 가능
3. 팝업 버튼:
    - [팬트리에 추가]
    - [추가 안 함]
4. 완료 처리 시:
    - 해당 장보기 대상 식사의 상태 → `장보기 완료`
    - 선택된 아이템만 팬트리에 반영
    - 리스트는 완료 상태가 되며 이후 read-only

### 공유 정책

- 공유 텍스트는 **구매 섹션 항목만 포함**
- 팬트리 제외 섹션 항목은 공유 대상에서 제외

### 상태

- Empty(구매 섹션이 비었을 때): “팬트리에 이미 있어서 장볼 재료가 없어요”
- Read-only 안내: “완료된 장보기 기록은 수정할 수 없어요”
